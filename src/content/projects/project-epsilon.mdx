## Problem

This case study is being written. Check back soon for the full story. Writing database migrations by hand is tedious and error-prone, especially when iterating on schema design. I wanted a visual tool that let me drag, connect, and export without leaving the browser.

## Solution & Approach

This case study is being written. Check back soon for the full story. The tool renders an interactive canvas where tables can be created, positioned, and connected with foreign key relationships. Exporting generates valid SQL migration files for either PostgreSQL or MySQL.

## Tech Stack Detail

This case study is being written. Check back soon for the full story. D3.js powers the interactive canvas with draggable nodes and relationship lines. React manages the UI state and property panels. TypeScript ensures the schema model stays consistent between the visual representation and the SQL generator.

## Challenges & Lessons

This case study is being written. Check back soon for the full story. The biggest challenge was generating correct SQL for two different database dialects. PostgreSQL and MySQL handle auto-increment, data types, and constraint syntax differently, which required a dialect abstraction layer.
