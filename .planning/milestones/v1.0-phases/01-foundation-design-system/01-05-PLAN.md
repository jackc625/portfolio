  <action>
Two targeted fixes in src/layouts/BaseLayout.astro:

**Fix 1 — Font import (UAT gap 1):**
Change line 2 from:
```
import { Font } from "astro:assets";
```
to:
```
import Font from "astro/components/Font.astro";
```
The Astro 6 Fonts API exports the Font component as a default export from `astro/components/Font.astro`, not as a named export from `astro:assets`. The current wrong import resolves to undefined, which means all three `<Font>` calls in the head are no-ops — no @font-face blocks are injected and no .woff2 files are requested.

**Fix 2 — Favicon link (UAT gap 2):**
Add the following line inside the `<head>` element, before the `<Font>` calls:
```html
<link rel="icon" href="/favicon.svg" type="image/svg+xml" />
```
The file `public/favicon.svg` already exists and contains the JC initials with dark background (#0e0f11) and teal text (#3db8a9), but the HTML never references it. Without a `<link rel="icon">` tag, browsers default to requesting `/favicon.ico` which does not exist.

Do NOT change anything else in the file. The rest of the layout (CSS import, Props interface, body classes, slot) is correct.
