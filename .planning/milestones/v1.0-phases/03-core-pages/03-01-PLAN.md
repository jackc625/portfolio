    The implementor MUST open https://shiyunlu.com in a browser and use DevTools (Elements panel > Computed Styles) to extract the exact color values used on the site. Key values to extract:
    - Background color of the main page body (likely a very dark navy/black)
    - Background color of card/secondary surfaces
    - Primary text color (light on dark)
    - Secondary/muted text colors
    - Accent color(s) used for links, highlights, interactive elements
    - Border/divider colors

    Convert extracted hex/rgb values to oklch format. Update these `:root` custom properties:
    - `--token-bg-primary`: near-black/dark navy background from shiyunlu.com body
    - `--token-bg-secondary`: slightly lighter surface color from shiyunlu.com cards/sections
    - `--token-accent`: primary accent color from shiyunlu.com (links, highlights)
    - `--token-accent-hover`: lighter/brighter variant of accent for hover states
    - `--token-text-primary`: main text color (light, near-white)
    - `--token-text-secondary`: secondary text color (muted, for descriptions)
    - `--token-text-muted`: very muted text (for labels, metadata)
    - `--token-border`: subtle border color
    - `--token-border-hover`: hover border color
    - Keep `--token-destructive`, `--token-success`, `--token-warning` as-is (semantic colors are not site-specific)

    **Step 4: Update typography scale tokens if needed**

    Review shiyunlu.com's type scale in DevTools. If the display/heading/body sizes differ significantly from current values, update:
    - `--token-text-sm`: small text size
    - `--token-text-base`: body text size
    - `--token-text-heading`: heading text size
    - `--token-text-display`: hero/display text size

    Keep the `clamp()` fluid scaling approach. Adjust the min/preferred/max values to match shiyunlu.com's scale.

    **Step 5: Update spacing tokens if needed**

    Review shiyunlu.com's spacing in DevTools (padding, margins, gaps). If the spacing scale differs from current values, update the `--token-space-*` variables. shiyunlu.com tends to use generous whitespace.

    **Step 6: Update the font-family fallback in @theme bridge**

    Since the new fonts are geometric sans-serif (not serif like Instrument Serif), update the `--font-display` fallback chain in both `@theme` and `@theme inline` blocks:
    ```css
    --font-display: var(--font-heading), ui-sans-serif, sans-serif;
    ```
    (Change `ui-serif, serif` to `ui-sans-serif, sans-serif` since Inter is a sans-serif font.)

    **CRITICAL: Do NOT change any `--color-*` mappings in the `@theme` block. Do NOT change `--color-bg-primary: var(--token-bg-primary)` etc. These stay exactly as-is.**
