    Add `@media print` rules at the END of the file (after all other styles). These rules implement D-20, D-21, and RESM-03.

    ```css
    /* ============================================
     * Print Stylesheet (RESM-03, D-20, D-21)
     * Scope: Resume page primarily. Strips chrome,
     * forces black-on-white, shows link URLs.
     * ============================================ */
    @media print {
      /* Hide non-content elements */
      header,
      footer,
      #mobile-menu,
      .skip-to-content,
      [data-no-print] {
        display: none !important;
      }

      /* Force black on white — overrides both dark and light themes */
      :root,
      [data-theme="light"] {
        --token-bg-primary: white;
        --token-bg-secondary: #f5f5f5;
        --token-text-primary: black;
        --token-text-secondary: #333;
        --token-text-muted: #666;
        --token-accent: #0066cc;
        --token-accent-hover: #0066cc;
        --token-border: #ccc;
        --token-border-hover: #999;
      }

      body {
        font-size: 11pt;
        line-height: 1.4;
        color: black;
        background: white;
      }

      /* Remove background colors/shadows from all elements */
      * {
        background: transparent !important;
        box-shadow: none !important;
        text-shadow: none !important;
      }

      /* Override specific backgrounds that need to stay */
      body {
        background: white !important;
      }

      /* Page break control */
      h2, h3 {
        break-after: avoid;
      }
      section {
        break-inside: avoid;
      }

      /* Remove max-width constraints for full-page printing */
      .max-w-3xl,
      .max-w-\\[90rem\\] {
        max-width: none;
      }

      /* Remove padding from main content area */
      main {
        padding-top: 0 !important;
      }

      /* Show link URLs for external links */
      a[href^="http"]::after {
        content: " (" attr(href) ")";
        font-size: 9pt;
        color: #666;
        font-family: var(--font-body);
      }

      /* Don't show URLs for internal nav links (they're hidden anyway) */
      header a::after,
      footer a::after,
      nav a::after {
        content: none !important;
      }

      /* Print-specific heading styles */
      h1 {
        font-size: 16pt;
        font-weight: 700;
        margin-bottom: 4pt;
      }
      h2 {
        font-size: 13pt;
        font-weight: 600;
        border-bottom: 1px solid #ccc;
        padding-bottom: 3pt;
      }

      /* Skill tags: plain text for printing */
      .font-mono[class*="border"] {
        border: none !important;
        padding: 0 !important;
        display: inline;
      }
      .font-mono[class*="border"]::after {
        content: ", ";
      }
      .font-mono[class*="border"]:last-child::after {
        content: "";
      }
    }
    ```

    ```astro

    ```

    **Modify `src/pages/index.astro`:**

    1. Add import: `import JsonLd from "../components/JsonLd.astro";`
    2. Add Person schema inside the `<BaseLayout>` slot, at the very top (before CanvasHero). Use the `<head>` slot if BaseLayout supports it, otherwise place inside body (JSON-LD is valid anywhere in the document per Google docs):

    ```astro
    <JsonLd schema={{
      "@context": "https://schema.org",
      "@type": "Person",
      "name": "Jack Cutrara",
      "jobTitle": "Software Engineer",
      "url": "https://jackcutrara.com",
      "sameAs": [
        "https://github.com/jackc625",
        "https://linkedin.com/in/jackcutrara"
      ]
    }} />
    ```

    Place this right after the opening `<BaseLayout ...>` tag, before `<CanvasHero>`.

    **Modify `src/pages/projects/[id].astro`:**

    1. Add import: `import JsonLd from "../../components/JsonLd.astro";`
    2. Build the CreativeWork schema dynamically from the `project` prop. Add right after the opening `<BaseLayout ...>` tag, before the first `<section>`:

    ```astro
    <JsonLd schema={{
      "@context": "https://schema.org",
      "@type": "CreativeWork",
      "name": project.data.title,
      "description": project.data.tagline,
      "url": `https://jackcutrara.com/projects/${project.id}`,
      "author": {
        "@type": "Person",
        "name": "Jack Cutrara"
      },
      "keywords": project.data.techStack.join(", "),
      ...(project.data.githubUrl ? {
        "codeRepository": project.data.githubUrl
      } : {}),
      ...(project.data.demoUrl ? {
        "url": project.data.demoUrl
      } : {})
    }} />
    ```

    Use `codeRepository` (Schema.org SoftwareSourceCode property, also valid on CreativeWork) for the GitHub URL since it's more specific than `sourceOrganization`. Only include `codeRepository` if `githubUrl` exists. Only include `demoUrl` if it exists.
