  <action>
    Present the human with verification steps covering all Phase 5 requirements. Start the dev server with `npx astro dev` and guide the user through each verification area.

    **What was built:**
    Complete dark/light mode system with theme toggle, GSAP scroll-triggered animations, page transitions, hover micro-interactions, enhanced canvas hero, print stylesheet, and JSON-LD structured data.

    **Verification steps for the human:**

    Start the dev server: `npx astro dev`

    **Theme System (DSGN-02, DSGN-03, DSGN-04):**
    1. Visit http://localhost:4321 — site should render in your OS-preferred theme (or dark by default)
    2. Click the theme toggle icon in the header — colors should smoothly crossfade (~300ms) to the other theme
    3. Verify light theme has warm off-white/cream backgrounds (not stark white)
    4. Verify light theme accent is a deeper, richer blue than dark theme
    5. Toggle back to dark — verify smooth transition
    6. Reload the page — theme should persist (no flash of wrong theme)
    7. On mobile width: verify toggle is visible between logo and hamburger (NOT inside mobile menu)
    8. Check all pages in light mode: Home, About, Projects, a case study, Resume, Contact
    9. Verify text is readable on all backgrounds in both themes (WCAG AA compliance)

    **Scroll Animations (ANIM-01, ANIM-04):**
    10. Visit /about — scroll down slowly. Sections should fade in with slight slide-up as they enter viewport
    11. Visit /projects — project cards should appear with staggered waterfall effect
    12. Visit /contact — contact channels should stagger in
    13. Page title headings should have line-by-line slide-up reveal (SplitText)
    14. Navigate to a different page and back — animations should replay

    **Page Transitions (ANIM-02):**
    15. Click any nav link — page should crossfade smoothly (old fades out, new fades in)
    16. Verify no jarring flash or layout jump during transition

    **Hover Interactions (ANIM-03):**
    17. Hover over a project card — should lift with subtle shadow
    18. Hover over desktop nav links — should see animated underline grow from left
    19. Hover over footer social icons — should translate up slightly
    20. Hover over theme toggle — icon should change color

    **Canvas Hero (D-24):**
    21. On home page, move mouse over the canvas — particles should gently respond to cursor
    22. Observe that some particles move slower/fainter (depth parallax effect)
    23. Overall motion should feel calmer and more flowing than before

    **Reduced Motion:**
    24. Enable "reduce motion" in your OS accessibility settings
    25. Reload — all content should be immediately visible with no animations
    26. Page transitions should be instant swaps (no crossfade)
    27. Hover transforms (card lift, icon lift) should be disabled; color changes still work

    **Print Stylesheet (RESM-03):**
    28. Visit /resume, press Ctrl+P (or Cmd+P)
    29. Print preview should show black text on white background
    30. No header/footer/nav visible in print
    31. "Jack Cutrara — Software Engineer" header with contact info should appear at top
    32. Download button should be hidden
    33. External link URLs should appear inline after links

    **JSON-LD (SEOA-03):**
    34. Right-click > View Page Source on home page — search for "application/ld+json"
    35. Verify Person schema with name, jobTitle, url, sameAs
    36. Check the same on /about
    37. Check any /projects/[id] — should have CreativeWork schema with project title and tech stack

    **Resume signal:** Type "approved" to proceed, or describe any issues that need fixing.
