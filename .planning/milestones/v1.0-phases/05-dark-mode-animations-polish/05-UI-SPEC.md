### Light Theme (NEW — D-16, D-17, D-18, D-19)

Direction: Warm off-white / cream backgrounds. Deeper, richer blue accent than dark mode's muted blue. Must independently pass WCAG AA (4.5:1+).

| Token | Light Value | Rationale |
|-------|-------------|-----------|
| --token-bg-primary | oklch(0.975 0.008 85) | Warm cream off-white (hue 85 = warm yellow undertone, not cool gray). L=0.975 avoids stark white. |
| --token-bg-secondary | oklch(0.94 0.01 85) | Slightly warmer/deeper cream for card surfaces and secondary areas. |
| --token-text-primary | oklch(0.13 0.01 270) | Near-black with slight warm cast. Contrast vs bg-primary: ~16:1 (passes AAA). |
| --token-text-secondary | oklch(0.38 0.015 270) | Dark gray for body text. Contrast vs bg-primary: ~8:1 (passes AA). |
| --token-text-muted | oklch(0.52 0.01 270) | Medium gray for labels and metadata. Contrast vs bg-primary: ~4.8:1 (passes AA). |
| --token-accent | oklch(0.48 0.18 250) | Deeper, more saturated blue than dark mode (D-17). Contrast vs bg-primary: ~5.5:1 (passes AA). |
| --token-accent-hover | oklch(0.40 0.20 250) | Even deeper blue on hover. Contrast vs bg-primary: ~7.5:1. |
| --token-border | oklch(0.88 0.006 85) | Warm light gray border. Subtle on cream backgrounds. |
| --token-border-hover | oklch(0.78 0.008 85) | Darker warm gray on hover. |
| --token-destructive | oklch(0.55 0.22 25) | Darker red for light backgrounds. Contrast vs bg-primary: ~5:1. |
| --token-success | oklch(0.50 0.15 155) | Darker green for light backgrounds. Contrast vs bg-primary: ~4.5:1. |
| --token-warning | oklch(0.55 0.15 85) | Darker amber for light backgrounds. |

**Canvas hero light mode (D-18):** Trail color changes from `rgba(10, 10, 14, 0.03)` to a warm cream equivalent `rgba(246, 243, 235, 0.03)`. Stroke color derived from computed --token-accent (already dynamic). The canvas reads accent from CSS tokens at init time — automatic adaptation when theme switches.

**WCAG AA verification approach:** All text-on-background combinations must achieve 4.5:1+ contrast ratio. The oklch lightness values are calibrated to meet this. Implementation must verify with a contrast checker tool.

### Accent Reserved-For (both themes)

1. Project card title text on hover (`group-hover:text-accent`)
2. CTA button text and underline (CTAButton.astro)
3. External link text (GitHub/Demo links on case studies)

6. Theme toggle icon color on hover
7. Nav link text for active page indicator (if changed from text-primary)

Accent is NEVER used for: backgrounds, section fills, body text, decorative borders at rest, or loading indicators.

## Theme System Contract

### Toggle Component

**Location:** Header.astro nav bar, right side, before hamburger trigger on mobile (D-01, D-05).

**Form:** Icon-only button, 40x40px touch target (D-02).

**Icon states:**
- Dark mode active: Sun icon (indicates "switch to light") — 20x20px SVG, `stroke="currentColor"` with stroke-width 1.5
- Light mode active: Moon icon (indicates "switch to dark") — 20x20px SVG, `stroke="currentColor"` with stroke-width 1.5

**Morph animation (D-03):** When toggling, the icon transitions with a combined rotation (180deg) and scale (0.8 to 1.0) animation over 300ms using `ease-out`. Sun rays can fade while moon crescent fades in (or vice versa). Implement via CSS transitions on two overlapping SVG elements with opacity crossfade, or via GSAP timeline if CSS alone is insufficient.

**Desktop placement:** After the last nav link in the `<ul>`, separated by `gap-8` (matching nav link spacing). Visually aligned with nav links.

**Mobile placement:** In the header bar, between the logo and the hamburger button. Always visible — NOT inside the mobile menu overlay (D-05).

**Accessibility:**
- `<button>` element with `aria-label="Switch to light mode"` / `"Switch to dark mode"` (updates dynamically)
- `focus-visible:ring-2 focus-visible:ring-accent focus-visible:outline-none focus-visible:rounded`
- Keyboard accessible via Tab + Enter/Space

### Theme Detection & Persistence (D-06)

**Inline `<head>` script (no-flash):**
1. Check `localStorage.getItem('theme')` for persisted preference
2. If no stored preference, check `window.matchMedia('(prefers-color-scheme: light)').matches`
3. If OS prefers light, set `document.documentElement.setAttribute('data-theme', 'light')`
4. If OS prefers dark or no preference, leave default (dark-first architecture — no attribute needed)
5. Script must be inline in `<head>` BEFORE any CSS renders — prevents flash of wrong theme

**Toggle behavior:**
1. Read current `data-theme` attribute
2. Toggle between removing attribute (dark) and setting `data-theme="light"`
3. Persist to `localStorage.setItem('theme', 'light' | 'dark')`
4. Re-initialize canvas hero if on home page (canvas reads accent color at init)

### Theme Transition (D-04)

**Cross-fade effect:** When theme switches, all color properties transition smoothly over 300ms.

```css
html {
  transition: background-color 300ms ease, color 300ms ease;
}
html *,
html *::before,
html *::after {
  transition: background-color 300ms ease, color 300ms ease, border-color 300ms ease, box-shadow 300ms ease;
}
```

**Exception:** Canvas hero does NOT cross-fade — it reinitializes with new colors. The brief canvas re-render is acceptable.

**`prefers-reduced-motion` override:** When reduced motion is preferred, set transition-duration to 0ms (instant switch, no animation).

## Animation Contract

### Global Animation Parameters

| Parameter | Value | Source |
|-----------|-------|--------|
| Scroll trigger start | "top 85%" (element top hits 85% of viewport) | Discretion |
| Scroll trigger end | not pinned — one-shot triggers | D-07 |
| Fade-in duration | 0.8s | Discretion |
| Slide-up distance | 24px (--token-space-lg) | Discretion |
| Easing | "power2.out" (GSAP) | Discretion |
| Stagger delay (cards) | 100ms (0.1s) between each card | D-09 |
| Stagger delay (SplitText lines) | 120ms (0.12s) between each line | Discretion |
| Replay on navigation | Yes — animations trigger every page load | D-12, D-15 |

### Animation Types

#### 1. Section Reveal (D-07, D-08)

**Applies to:** Every `<section>` on every page.

**Initial state:** `opacity: 0; transform: translateY(24px)`

**Animated state:** `opacity: 1; transform: translateY(0)`

**Trigger:** ScrollTrigger, element enters viewport at 85% scroll position.

**Duration:** 0.8s with "power2.out" easing.

**Implementation:** Register a GSAP ScrollTrigger per section. On `astro:page-load`, query all `[data-animate="section"]` elements and create triggers. On `astro:before-preparation`, kill all triggers.

#### 2. Card Stagger (D-09)

**Applies to:** Project cards on `/projects` page, skill groups on `/about` page.

**Initial state:** `opacity: 0; transform: translateY(24px)`

**Animated state:** `opacity: 1; transform: translateY(0)`

**Stagger:** 100ms between each card, triggered when the grid container enters viewport.

**Duration:** 0.6s per card with "power2.out" easing.

**Implementation:** ScrollTrigger on the grid container, `gsap.from()` with `stagger: 0.1` on child elements marked `[data-animate="stagger-item"]`.

#### 3. Display Heading SplitText (D-11)

**Applies to:** Hero heading on home page, page title headings on About, Projects, Resume, Contact pages. Only elements with `[data-animate="split-text"]`.

**Initial state:** Each line: `opacity: 0; transform: translateY(100%); clip-path: inset(0 0 100% 0)`

**Animated state:** Each line: `opacity: 1; transform: translateY(0); clip-path: inset(0 0 0 0)`

**Stagger:** 120ms between each line.

**Duration:** 0.7s per line with "power2.out" easing.

**Implementation:** GSAP SplitText splits text by lines. Wrap each line in a clip container. Animate lines with stagger. Re-split on resize (SplitText.revert() + re-split).

**Fallback:** If SplitText fails to load or text is single-line, fall back to standard section reveal (fade + slide-up).

#### 4. Hover Micro-Interactions (D-10)

**Project cards:**
| State | Effect |
|-------|--------|
| Default | No shadow, no transform |
| Hover | `box-shadow: 0 4px 20px rgba(0, 0, 0, 0.15)` + `transform: translateY(-2px)` over 300ms ease |
| Hover (title) | `color: var(--token-accent)` over 200ms (existing) |
| Hover (thumbnail) | `transform: scale(1.02)` over 300ms (existing) |

**Navigation links:**
| State | Effect |
|-------|--------|
| Default | No underline decoration |
| Hover | Animated underline from left to right: `background-size` transition from `0% 1px` to `100% 1px` at bottom, using `background-image: linear-gradient(currentColor, currentColor)` over 300ms ease |

**CTA / external links:**
| State | Effect |
|-------|--------|
| Default | `border-bottom: 1px solid` at 30% opacity (existing) |
| Hover | Border opacity transitions to 100% (existing), slight letter-spacing increase `+0.01em` over 200ms |

**Theme toggle:**
| State | Effect |
|-------|--------|
| Default | `text-text-muted` |
| Hover | `text-text-primary` + subtle rotation (15deg clockwise for sun, -15deg for moon) over 200ms |

**Footer social icons:**
| State | Effect |
|-------|--------|
| Default | `text-text-muted` |
| Hover | `text-text-primary` + `transform: translateY(-2px)` over 200ms |

#### 5. Canvas Hero Enhancements (D-24)

**Mouse influence:**
- Track mouse position via `mousemove` event on canvas container
- Particles within 150px radius of cursor are gently pushed: angle offset of `+0.3 radians` toward cursor direction
- Influence falls off linearly from center to edge of radius
- On touch devices: no mouse influence (touch events not tracked for performance)

**Parallax depth:**
- Assign each particle a `depth` value: 0.5, 0.75, or 1.0 (3 layers)
- Multiply particle speed by depth value: deeper particles (0.5) move slower
- Multiply particle stroke opacity by depth: deeper particles are fainter (`opacity * depth`)
- Creates visual depth without actual 3D rendering

**Parameter tuning:**
- Reduce `noiseScale` from 0.003 to 0.0025 (smoother, more flowing curves)
- Reduce `speed` from 1.5 to 1.2 (calmer, more meditative feel)
- Increase stroke opacity from 0.15 to 0.18 (slightly more visible trails)
- Line width varies by depth: 0.3 (far), 0.4 (mid), 0.6 (near)

### Reduced Motion (D-13, ANIM-04)

When `prefers-reduced-motion: reduce` is detected:

1. **All GSAP animations:** Do not register. Show content in final state immediately (opacity: 1, translateY: 0).
2. **SplitText:** Do not split. Display heading as-is.
3. **Hover micro-interactions:** Keep color transitions (these are not motion). Remove all `transform` transitions (translateY, scale, rotate).
4. **Page transitions:** Instant swap, no crossfade.
5. **Canvas hero:** Static frame (already implemented).
6. **Theme toggle morph:** Instant icon swap, no rotation/scale animation.

**Implementation:** Check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` before any GSAP initialization. If true, set all `[data-animate]` elements to their final visible state and return without creating ScrollTriggers.

## Page Transition Contract (D-14, ANIM-02)

**Mechanism:** Astro View Transitions API via `<ClientRouter />` (already in BaseLayout.astro).

**Transition style:** Crossfade. Old page fades out over 200ms, new page fades in over 200ms. Total perceived transition: ~250ms with overlap.

**CSS:**
```css
::view-transition-old(root) {
  animation: fade-out 200ms ease-in forwards;
}
::view-transition-new(root) {
  animation: fade-in 200ms ease-out forwards;
}
@keyframes fade-out {
  to { opacity: 0; }
}
@keyframes fade-in {
  from { opacity: 0; }
}
```

**GSAP cleanup on navigation (D-15):**
- `astro:before-preparation`: Kill all ScrollTrigger instances, revert all SplitText instances, cancel all GSAP tweens.
- `astro:page-load`: Re-query DOM, re-initialize all animations for the new page.

**Reduced motion:** `::view-transition-old(root)` and `::view-transition-new(root)` get `animation: none` when `prefers-reduced-motion: reduce`.

## GSAP Lifecycle Pattern

All GSAP code in this project must follow this lifecycle pattern for View Transitions compatibility:

```
// Pattern for every page/component with GSAP animations
let triggers: ScrollTrigger[] = [];
let splits: SplitText[] = [];

function initAnimations() {
  // 1. Register plugins
  gsap.registerPlugin(ScrollTrigger, SplitText);

  // 2. Check reduced motion
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
    // Show all content, return early
    gsap.set('[data-animate]', { opacity: 1, y: 0 });
    return;
  }

  // 3. Create animations, store references
  // ... ScrollTrigger.create() -> push to triggers[]
  // ... new SplitText() -> push to splits[]
}

function cleanupAnimations() {
  triggers.forEach(t => t.kill());
  triggers = [];
  splits.forEach(s => s.revert());
  splits = [];
  ScrollTrigger.getAll().forEach(t => t.kill());
}

document.addEventListener('astro:page-load', initAnimations);
document.addEventListener('astro:before-preparation', cleanupAnimations);
```

**Source:** CanvasHero.astro existing lifecycle pattern, CLAUDE.md GSAP+Astro View Transitions notes
