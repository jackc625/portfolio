   **Idempotent initialization guard (addresses review concern from Claude+Codex):**
   When `transition:persist` preserves the DOM but JS may re-execute on `astro:page-load`,
   duplicate event handlers would fire. The initialization MUST be idempotent.
   ```typescript
   // Module-level initialization guard — prevents duplicate handlers
   // across view transition navigations where transition:persist preserves
   // the DOM but astro:page-load re-fires the script.
   // Addresses review concern: Claude MEDIUM, Codex MEDIUM — duplicate handler risk.
   let chatInitialized = false;

   document.addEventListener("astro:page-load", () => {
     const panel = document.getElementById("chat-panel");
     if (!panel) return; // Widget not in DOM yet

     // Idempotency: if already initialized AND the same DOM elements exist,
     // skip re-initialization. The transition:persist keeps our state alive.
     if (chatInitialized && panel.dataset.chatBound === "true") return;

     // Mark as initialized
     chatInitialized = true;
     panel.dataset.chatBound = "true";

     initChat(); // Main initialization function
   });
   ```

   **Transition:persist state handling (addresses review concern from Codex HIGH):**
   When navigating with an active stream, the DOM persists but fetch/ReadableStream
   state needs explicit handling:
   ```typescript
   // On astro:before-preparation, if a stream is active:

   // - The streaming fetch is a Promise held in closure scope — it continues
   ```

   **Focus trap with dynamic element re-querying (D-33):**
   Addresses review concern: Claude MEDIUM — focus trap must re-query focusable elements
   on EACH Tab keypress because bot messages dynamically add `<a>` links and copy buttons.
   Static query at panel-open time would miss these elements.
   ```typescript
   let cleanupFocusTrap: (() => void) | null = null;

   function setupFocusTrap(panel: HTMLElement): () => void {
     const handler = (e: KeyboardEvent) => {
       if (e.key === "Escape") {
         closePanel();
         return;
       }
       if (e.key !== "Tab") return;

       // Re-query focusable elements on EACH Tab keypress — bot messages
       // dynamically add <a> links and copy buttons that must be included.
       // Addresses review concern: Claude MEDIUM — static query misses dynamic elements.
       const focusable = panel.querySelectorAll<HTMLElement>(
         'button:not([disabled]), [href], input:not([disabled]), select:not([disabled]), textarea:not([disabled]), [tabindex]:not([tabindex="-1"])'
       );
       if (focusable.length === 0) return;
       const first = focusable[0];
       const last = focusable[focusable.length - 1];
       if (e.shiftKey && document.activeElement === first) {
         e.preventDefault();
         last.focus();
       } else if (!e.shiftKey && document.activeElement === last) {
         e.preventDefault();
         first.focus();
       }
     };
     panel.addEventListener("keydown", handler);
     return () => panel.removeEventListener("keydown", handler);
   }
   ```
   Call `setupFocusTrap(panel)` when panel opens, store the cleanup function.
   Call `cleanupFocusTrap()` when panel closes AND on `astro:before-preparation`.

   **Focus management:**
   - When panel opens: `cleanupFocusTrap = setupFocusTrap(panel)`, then focus textarea
   - When panel closes: call `cleanupFocusTrap()`, set to null, return focus to bubble
   - Escape key in handler closes panel (via `closePanel()` which handles cleanup)
