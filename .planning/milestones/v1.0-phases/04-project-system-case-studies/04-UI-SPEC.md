| 3xl | 64px (4rem) | Asymmetric grid column gap (matching About page pattern) |

Exceptions: none

**Source:** global.css :root spacing tokens (Phase 1), established usage in about.astro (Phase 3)

## Typography

| Role | Size Token | Weight | Line Height | Usage in Phase 4 |
|------|------------|--------|-------------|-------------------|
| Body | --token-text-base (clamp 15px-17px) | 400 (regular) | 1.6 (body default) / 1.7 (prose paragraphs) | Case study body text, project descriptions |
| Label | --token-text-sm (clamp 12px-14px) | 400 (regular) | 1.4 | Mono uppercase section labels, tech tags, image captions, "Next Project" label |
| Heading | --token-text-heading (clamp 20px-32px) | 400 (regular) | 1.5 | Case study section content lead-in, project card titles |
| Display | --token-text-display (clamp 40px-88px) | 400 (regular) | 0.95 | Projects page title ("Things I've built."), case study project title |

**Font-weight constraint:** Only weight 400 (regular) used in this phase. No bold/semibold in the editorial design language — hierarchy comes from size, color, and spacing, not weight. This matches the shiyunlu.com pattern established in Phase 3.

**Font-family mapping:**
- `font-display` (Inter via --font-heading): Display and heading roles
- `font-body` (Inter via --font-sans): Body text
- `font-mono` (IBM Plex Mono via --font-code): Labels, tech tags, captions, metadata

**Source:** global.css typography tokens, about.astro established patterns, astro.config.mjs font weights

## Color

| Role | Token | Value | Usage in Phase 4 |
|------|-------|-------|-------------------|
| Dominant (60%) | --token-bg-primary | oklch(0.1 0.005 270) | Page backgrounds, card interiors (no separate card bg) |
| Secondary (30%) | --token-bg-secondary | oklch(0.15 0.005 270) | Thumbnail placeholder cards, next-project navigation background, hover card backgrounds |
| Accent (10%) | --token-accent | oklch(0.7 0.1 250) | See reserved-for list below |
| Destructive | --token-destructive | oklch(0.63 0.2 25) | Not used in Phase 4 (no destructive actions) |

**Text colors:**

| Token | Value | Usage |
|-------|-------|-------|
| --token-text-primary | oklch(0.95 0.005 270) | Titles, project names, body text on bg-primary |
| --token-text-secondary | oklch(0.7 0.01 270) | Taglines, secondary descriptions, prose paragraphs |
| --token-text-muted | oklch(0.5 0.01 270) | Mono labels, tech tags, image captions, metadata |

**Border colors:**

| Token | Value | Usage |
|-------|-------|-------|
| --token-border | oklch(0.22 0.005 270) | Editorial list item bottom borders, card borders, section dividers |
| --token-border-hover | oklch(0.32 0.005 270) | Hover state on editorial list borders, card borders |

**Accent reserved for (explicit list):**
1. Project card title text on hover (`group-hover:text-accent`)
2. GitHub/Demo link text and underline on case study pages
3. "Next Project" arrow icon and hover text
4. Focus-visible outline rings on all interactive elements
5. CTA-style links (e.g., "View Project" if used)

Accent is NEVER used for: backgrounds, card fills, section backgrounds, borders at rest, body text, or decorative elements.

**Source:** global.css color tokens, FeaturedProjectItem.astro hover pattern, CTAButton.astro accent usage

## Component Inventory

### New Components (Phase 4)

#### 1. ProjectCard.astro
**Purpose:** Featured project card with thumbnail, title, tagline, and tech tags. Links to case study page.
**Props:** `project` (content collection entry)

**Visual contract:**
- Container: `<a>` tag wrapping entire card for clickability. `block` display.
- Thumbnail area: 16:9 aspect ratio (`aspect-video`). If `thumbnail` exists, render via Astro `<Image />` with `object-cover`. If absent, render `bg-bg-secondary` with centered project title in `font-display text-text-muted`.
- Content area below thumbnail: `padding` of `--token-space-lg` (24px) on all sides.
- Title: `font-display text-[length:var(--token-text-heading)] text-text-primary`. On card hover: `text-accent` with `transition-colors duration-200`.
- Tagline: `font-body text-[length:var(--token-text-sm)] text-text-secondary`. Single line, truncated with `line-clamp-1` if needed.
- Tech tags: `font-mono text-[length:var(--token-text-sm)] text-text-muted`. Comma-separated inline text (matching FeaturedProjectItem pattern, not pill chips — keeps cards clean).
- Card border: none at rest. On hover: subtle `border-border-hover` bottom border OR no border (cards distinguished by thumbnail area). Keep minimal — no card shadows, no rounded corners larger than `rounded` (4px).
- Focus-visible: `outline-2 outline-accent outline-offset-4 rounded`.

**Responsive:**
- Desktop (lg+): 3-column grid for 3 featured cards
- Tablet (md): 2-column grid
- Mobile: Single column, full-width stacked (D-23)

#### 2. CaseStudySection.astro
**Purpose:** Reusable asymmetric grid section for case study pages. Mono uppercase label left, content right.
**Props:** `label` (string)

**Visual contract:**
- Outer: `<section aria-label={label}>` with padding `pb-[var(--token-space-section)] px-6 md:px-10 lg:px-16`
- Inner: `max-w-[90rem] mx-auto grid grid-cols-1 lg:grid-cols-[1fr_2fr] gap-[var(--token-space-3xl)]`
- Label column: `font-mono text-[length:var(--token-text-sm)] text-text-muted uppercase tracking-[0.08em]`
- Content column: `max-w-2xl` container. Content via `<slot />`.
- On mobile (below lg): stacks to single column, label above content.

**Source:** Exact match of about.astro sections (Phase 3 inner page pattern, D-09)

#### 3. ArticleImage.astro
**Purpose:** Custom MDX image component. Wraps images in `<figure>` with optional `<figcaption>`.
**Props:** `src` (string | ImageMetadata), `alt` (string)

**Visual contract:**
- Outer: `<figure>` with vertical margin `my-[var(--token-space-xl)]` (32px above and below)
- Image: `w-full rounded aspect-video object-cover`. Lazy loading via `loading="lazy"`.
- If `src` is ImageMetadata (local import): render via Astro `<Image />` for optimization.
- If `src` is string (external URL): render standard `<img>` tag.
- Caption (figcaption): Render when `alt` text is present. `font-mono text-[length:var(--token-text-sm)] text-text-muted mt-[var(--token-space-sm)]`. Left-aligned.
- No border, no shadow, no background behind image.

**Source:** D-14 (image captions), RESEARCH.md Pattern 3

#### 4. NextProject.astro
**Purpose:** Navigation link at the bottom of case study pages, linking to the next project.
**Props:** `project` (content collection entry for next project)

**Visual contract:**
- Outer: `<section>` with `bg-bg-secondary` background. Full-width band. Padding: `py-[var(--token-space-section)] px-6 md:px-10 lg:px-16`.
- Inner: `max-w-[90rem] mx-auto`. Content is a single `<a>` link.
- Layout: Label on top, project title below, optional arrow indicator.
- Label: `font-mono text-[length:var(--token-text-sm)] text-text-muted uppercase tracking-[0.08em]` reading "Next Project".
- Title: `font-display text-[length:var(--token-text-heading)] text-text-primary`. On hover: `text-accent` with `transition-colors duration-200`.
- Arrow: Right-pointing arrow (inline SVG, 24x24, `stroke="currentColor"`) after the title. On hover: translate-x by 4px (`group-hover:translate-x-1 transition-transform duration-200`).
- Focus-visible: `outline-2 outline-accent outline-offset-4 rounded`.
- The entire section is wrapped in an `<a>` tag — clicking anywhere navigates.

**Source:** D-10 (next project navigation), shiyunlu.com project browsing flow

#### 7. projects/[id].astro (new — case study page)
**Layout structure (top to bottom):**

1. **Hero section:**
   - Padding: `pt-[var(--token-space-section)] pb-[var(--token-space-2xl)] px-6 md:px-10 lg:px-16`
   - Container: `max-w-[90rem] mx-auto`
   - Back link: Small mono text "Back to Projects" with left arrow, `text-text-muted hover:text-accent`. Top of hero.
   - Title: `font-display text-[length:var(--token-text-display)] tracking-[-0.03em] leading-[0.95]` showing project title.
   - Tagline: `font-body text-[length:var(--token-text-heading)] text-text-secondary mt-[var(--token-space-md)]` showing project tagline.
   - Tech tags row: `font-mono text-[length:var(--token-text-sm)] text-text-muted mt-[var(--token-space-lg)]` comma-separated tech stack.
   - Links row (if githubUrl or demoUrl exist): `mt-[var(--token-space-lg)]`. Render as inline links with accent color: "View Source" and "Live Demo". Use `font-body text-[length:var(--token-text-sm)] text-accent hover:text-accent-hover uppercase tracking-[0.08em] border-b border-accent/30 hover:border-accent` (matching CTAButton pattern). Space between links: `gap-[var(--token-space-xl)]`.

2. **Hero image (if thumbnail exists):**
   - `mt-[var(--token-space-2xl)]` below links row
   - Full-width within container: `max-w-[90rem] mx-auto px-6 md:px-10 lg:px-16`
   - Image: `w-full aspect-video object-cover rounded` via Astro `<Image />` (D-13: 16:9 landscape)
   - If no thumbnail: omit entirely (no placeholder on case study pages — fallback only on project cards)

3. **MDX body content:**
   - Padding: `py-[var(--token-space-section)] px-6 md:px-10 lg:px-16`
   - Container: `max-w-[90rem] mx-auto`
   - MDX rendered via `<Content components={{ img: ArticleImage }} />`
   - H2 headings in MDX styled as section labels: `font-mono text-[length:var(--token-text-sm)] text-text-muted uppercase tracking-[0.08em]` with `mt-[var(--token-space-section)] mb-[var(--token-space-lg)]` (matching asymmetric section label pattern)
   - Body paragraphs: `font-body text-[length:var(--token-text-base)] text-text-secondary leading-[1.7] mb-[var(--token-space-lg)]`
   - The MDX body content renders inside a max-w-2xl prose container within the right column of the asymmetric grid, OR as a single constrained column. Recommendation: use single constrained-width column (`max-w-3xl`) for the MDX body to keep it simple. The asymmetric grid applies via `CaseStudySection` wrapper components IF needed, but MDX H2-based sections are simpler (RESEARCH.md Open Question 2 recommendation).
   - Prose styling for MDX rendered content (applied via Tailwind `prose` class or manual styles on the article container):
     - `h2`: styled as mono uppercase labels (see above)
     - `p`: text-text-secondary, 1.7 line-height, mb-lg spacing
     - `strong`: text-text-primary (promote to primary color for emphasis)
     - `a`: text-accent, hover:text-accent-hover, underline
     - `ul/ol`: text-text-secondary, ml-lg, mb-lg
     - `li`: mb-sm spacing between items
     - `blockquote`: border-l-2 border-accent/30, pl-lg, italic, text-text-muted

4. **Next project section:**
   - `NextProject` component (see Component Inventory above)
   - Full-width `bg-bg-secondary` band at the very bottom of the page, before footer

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All project cards and list items are `<a>` tags — natively keyboard-accessible via Tab + Enter. |
| Focus indicators | Every interactive element has `focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-4`. |
| Heading hierarchy | Projects page: h1 (page title). Case study page: h1 (project title), h2 (section headings in MDX body). No heading level skips. |
| Landmarks | Each section has `aria-label` attribute describing its content. |
| Image alt text | All `<Image />` components require alt text. ArticleImage uses alt for both accessibility and caption. Thumbnails: "Project name thumbnail" alt text. |
| Color contrast | All text/background combinations meet WCAG AA (4.5:1+). Verified by existing token choices from Phase 1 audit. |
| Link purpose | Links clearly describe destination: "View Source", "Live Demo", "Back to Projects", project titles as link text. |
