**CRITICAL: Use `entry.id` not `entry.slug`** -- Astro 6 Content Layer API uses `id` (per RESEARCH.md Pitfall 1).

**Frontmatter script (between the Astro fence delimiters):**

Imports:
- `BaseLayout` from `../../layouts/BaseLayout.astro`
- `ArticleImage` from `../../components/ArticleImage.astro`
- `NextProject` from `../../components/NextProject.astro`
- `{ getCollection, render }` from `astro:content`
- `{ Image }` from `astro:assets`

Export `getStaticPaths()`:
- `const projects = await getCollection("projects");`
- `const sorted = projects.sort((a, b) => a.data.order - b.data.order);`
- Return `sorted.map((project, index) => { const nextProject = sorted[(index + 1) % sorted.length]; return { params: { id: project.id }, props: { project, nextProject } }; })`

Page props:
- `const { project, nextProject } = Astro.props;`
- `const { Content } = await render(project);`
