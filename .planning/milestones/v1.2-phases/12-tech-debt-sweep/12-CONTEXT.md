<decisions>
## Implementation Decisions

### Copy-button parity (DEBT-04)
- **D-01:** Canonical copy-button markup = mono `COPY` → `COPIED` text label with accent-color flip on copy success. The SVG clipboard icon currently at `src/scripts/chat.ts:302` is replaced with the label. Rationale: mono label matches the editorial vocabulary already used across the site (SectionHeader counts, WorkRow meta, chat char count); picking SVG would fight the editorial system, and the replayed path already has the nicer behavior.
- **D-02:** Both creation paths — live-stream (`chat.ts:285-308`) and localStorage replay (`chat.ts:553-569`) — call a single shared helper (e.g. `renderCopyButton(content, container)`) so markup, classes, inline styles, aria-label, event wiring, and post-click label/color transitions are guaranteed identical. Dedup the code first, then pick the visual.
- **D-03:** Always-visible (no hover-reveal). Consistent with touch devices and with the existing replayed behavior.

### Build / lint warnings (DEBT-01)
- **D-04:** **Lightning-css is already clean.** Phase 9's `@source not .planning/` directive silenced all 4 warnings the v1.1 audit cited. Phase 12 scope pivots to the warnings `pnpm build` actually emits today: `wrangler.jsonc: Unexpected fields found in top-level field: "rate_limits"` (emitted ×6 per build) and an informational upgrade notice (Wrangler 4.80 → 4.83).
- **D-05:** Upgrade Wrangler to 4.83+ first, then re-run the build. If 4.83 recognizes `rate_limits`, the warning resolves for free. If 4.83 still flags it, inspect the current Cloudflare Workers config schema and either relocate `rate_limits` to the right nested location or remove the block (we're already rate-limiting in application code via `api/chat.ts`).
- **D-06:** Clean up the stale `px-[var(--token-*)]` literal inside the comment at `src/styles/global.css:34` — rewrite the comment without bracket-syntax literals so the `@source not` exclusion isn't load-bearing for that string.
- **D-07:** ESLint today emits 3 `unused-disable-directive` warnings:
  - `src/components/primitives/Container.astro:18` — delete the stale disable comment.
  - `worker-configuration.d.ts:9615, 9631` — generated by `wrangler types`; add `worker-configuration.d.ts` to ESLint `ignorePatterns` rather than editing the generated file (any `--fix` edit would be wiped on next `wrangler types` run).
- **D-08:** Zero-warning bar applies to: `pnpm build` (Astro + Vite + Wrangler output), `pnpm exec astro check` (already clean), `pnpm lint` (after D-07 fixes). All three must exit cleanly with no warnings before the phase is marked complete.

### MobileMenu inert strategy (DEBT-02)
- **D-09:** The `inert` attribute on `<header>`/`<main>`/`<footer>` while the menu is open is already implemented at `src/components/primitives/MobileMenu.astro:267-273` and removed at `:286-288`. Phase 12 extends this to cover `<ChatWidget />`, which is a body-level sibling and currently remains tabbable/clickable behind the menu backdrop. Add `inert` set/remove on the chat widget root in the same `openMenu()`/`closeMenu()` blocks.
- **D-10:** Keep the focus-trap keydown handler (`MobileMenu.astro:293-317`) as a belt-and-suspenders fallback. `inert` has 95%+ browser support in 2026 but older mobile Safari (<15.5) and Firefox (<112) lack it; the per-Tab re-query pattern is also a D-08 invariant shared with `src/scripts/chat.ts setupFocusTrap`. Keeping both preserves the documented invariant and stays resilient to old-browser edge cases.
- **D-11:** Verification is a scripted keyboard-cycle test, documented as a manual test step in `12-VALIDATION.md`. Steps: open menu on `/projects` (many background interactive elements — 6 WorkRows, footer social links, chat bubble); with DevTools console monitoring `document.activeElement`, press Tab 30+ times; assert `document.activeElement` never leaves the `.mobile-menu` subtree. Repeat with Shift+Tab. No new test-runner dependency.
- **D-12:** Extending inert to ChatWidget touches `chat.ts`-adjacent behavior, so the D-26 Chat Regression Gate runs as part of phase verification: XSS sanitization, exact-origin CORS, 5/60s rate limit, 30s AbortController timeout, focus trap re-query, localStorage persistence (50-msg cap / 24h TTL), SSE streaming with `async:false`, DOMPurify strict markdown, and clipboard idempotency (JS boolean + DOM data-attribute) all confirmed unchanged.

### Accepted trade-off documentation (DEBT-05, DEBT-06)
- **D-13:** Introduce a new `MASTER.md §2.4 Accepted token exceptions` subsection immediately after §2.3 Lock contract. Symmetrical with the existing §10 Chat Bubble Exception pattern — one grep'able home for future auditors.
- **D-14:** Two entries under §2.4:
  1. **`--ink-faint` contrast exception.** Token: `--ink-faint` (#A1A1AA) on `--bg` (#FAFAF7) computes 2.5:1, below WCAG AA 4.5:1 for normal text. Usage scope: tertiary/decorative metadata only — section counts, year labels, work stack labels, footer copy, next-project label, contact separator, chat char count. Rationale: editorial hierarchy requires visually-quieter tertiary text than 4.5:1 permits; primary text (`--ink` 18.9:1) and secondary (`--ink-muted` 7.4:1) pass comfortably. Lighthouse Accessibility lands at 95/100 (not 100) as a consequence — QUAL-01 threshold (≥90) still met.
  2. **Print stylesheet `#666`.** Color: `#666` literal at `global.css:199` inside `a[href^="http"]::after` URL annotations. Outside the 6-hex palette. Rationale: print-only, never reaches the screen; design tokens aren't required to extend into the print cascade where the editorial color system doesn't apply.
- **D-15:** MASTER.md only — no inline CSS comments duplicated at usage sites. Keeps single source of truth; avoids drift.
- **D-16:** Status: **permanent-accepted, will not be addressed in v1.x**. Each §2.4 entry includes this status line with rationale. Re-evaluation deferred to any future milestone that revisits the editorial color system.

### OG URL builder (DEBT-03)

Confirm each response carries a correct absolute URL with no double-origin corruption. Record URLs tested + results table in `12-VALIDATION.md`.

<code_context>
## Existing Code Insights

### Reusable Assets
- **Shared focus-trap pattern** — `MobileMenu.astro:293-317` and `chat.ts setupFocusTrap` both use the per-Tab re-query pattern (D-08). Any new focus-management code reuses the same shape.
- **Token system** — 6 hex custom properties in `global.css:9-14` are the only palette. New code consumes them via `var(--token)`.
- **Inert set/remove pattern** — already established in `MobileMenu.astro openMenu()` / `closeMenu()`. Extending it to ChatWidget is additive.

### Established Patterns
- **Scoped `<style>` in primitives** (MASTER.md D-03) — no Tailwind utilities inside primitive markup. CSS-only, scoped per-component.
- **Astro 6 Fonts API** via `astro.config.mjs` + `:root { --font-*-src: … }` + `@theme { --font-*: var(--font-*-src), fallback }`. Two-layer naming prevents self-reference bug. Don't touch.
- **`@source not` directive** silencing `.planning/` + `design-system/` from Tailwind template detection (`global.css:32-46`). Load-bearing — any replacement strategy must preserve this exclusion.
- **Dual-path chat message rendering** (live-stream vs replay) — both paths create DOM elements imperatively in `chat.ts`, not via Astro components. Normalization goes in the script itself.

### Integration Points
- `BaseLayout.astro` wraps every page — `<Header />`, `<MobileMenu />`, `<main>`, `<Footer />`, `<ChatWidget />` all mount here. Any inert-scope change is a single edit in `MobileMenu.astro`'s openMenu/closeMenu.
- `wrangler.jsonc` drives both dev and prod behavior — upgrade must preserve current `rate_limits` intent (which itself may be a no-op if 4.80 was already ignoring it).
- `eslint.config.*` governs what `pnpm lint` checks — adding an ignore for the generated `worker-configuration.d.ts` is the only config surface this phase touches.
