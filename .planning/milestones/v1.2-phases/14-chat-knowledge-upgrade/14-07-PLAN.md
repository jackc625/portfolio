<objective>
Close the only major UAT gap from Phase 14 UAT test 3 — chat replies truncating mid-section on verbose project questions (SeatWatch demonstrated; Daytrade / Clipify / Optimize AI equally at risk given their ~8-11k token source footprint). Root cause per `.planning/debug/chat-response-truncation.md`: the `max_tokens: 768` ceiling in `src/prompts/chat-request-shape.ts:33` is too low for grounded multi-paragraph project answers, AND the SSE consumer in `src/pages/api/chat.ts:104-118` only inspects `content_block_delta` events — it silently discards `message_delta` events whose `delta.stop_reason` carries `"max_tokens"`, so truncation ships as a clean `[DONE]` with no operator or visitor visibility.

Purpose: Two-part surgical fix — (1) raise the output budget to 1500 tokens (~1125 words, comfortable ceiling for a 2-4 paragraph deep-dive per the system-prompt's existing length guidance; 6.5× under Haiku 4.5's 8192 hard cap), and (2) add a `message_delta` branch to the stream consumer that both server-logs the truncation AND emits a diagnostic `{"truncated":true}` SSE frame before `[DONE]`. Preserves every byte of the Phase 7 D-26 regression surface and every byte of the CHAT-05 cache_control invariant. Updates the single-literal assertion in `tests/api/chat.test.ts` from `768` to `1500`, adds one new test proving the truncation frame is emitted, and patches 14-VERIFICATION.md to reflect the new literal.

Output: 4 modified files (chat-request-shape.ts, chat.ts, chat.test.ts, 14-VERIFICATION.md). Zero new files. Zero new dependencies. Zero SDK version bump. The prompt-adherence heading-emission drift ("Why It Matters") called out in the debug session is NOT in scope here — that's a separate prompt-rewrite, not a truncation fix.

```ts
let truncated = false;

for await (const event of response) {
  if (
    event.type === "content_block_delta" &&
    event.delta.type === "text_delta"
  ) {
    controller.enqueue(
      encoder.encode(
        `data: ${JSON.stringify({ text: event.delta.text })}\n\n`
      )
    );
  } else if (event.type === "message_delta") {
    // Anthropic signals final stop reason here. "max_tokens" means the model
    // hit the output-token ceiling mid-generation and the reply is clipped.
    // Log server-side for observability and emit a diagnostic SSE frame so
    // the client can surface a truncation hint (Phase 15 ANAL-03 will wire
    // the log line to the observability backend).
    if (event.delta.stop_reason === "max_tokens") {
      truncated = true;
      console.warn("chat.truncated", { stop_reason: "max_tokens" });
    }
  }
}

if (truncated) {
  controller.enqueue(encoder.encode(`data: ${JSON.stringify({ truncated: true })}\n\n`));
}
controller.enqueue(encoder.encode("data: [DONE]\n\n"));
controller.close();
```
