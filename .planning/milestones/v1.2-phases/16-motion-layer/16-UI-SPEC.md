# Phase 16 — UI Design Contract

> Visual and interaction contract for the Motion Layer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase character:** This phase does NOT introduce new visual primitives, colors, type roles, or copy. The v1.1 editorial system (MASTER.md §2-§5, §7-§10) is the locked visual contract. Phase 16 layers a single new dimension — **motion** — on top of that locked surface. The bulk of this contract is therefore the Motion section. Spacing, Typography, Color, and Copywriting are restated as inherited locks (no change permitted) so the planner and executor have a single authoritative read.

## Motion Contract (PHASE-16 CORE)

This is the substantive new contract Phase 16 adds. Every animation below is **gated by `@media (prefers-reduced-motion: no-preference)` or paired with an explicit `reduce` override** per MOTN-08 / D-23 / D-24.

### Property whitelist

Animations may move only these CSS properties:

| Property | Allowed | Why |
|----------|---------|-----|
| `transform` (translate, scale) | YES | GPU-composited, no layout/paint cost |
| `opacity` | YES | GPU-composited, no layout/paint cost |
| `box-shadow` | YES (chat bubble pulse only) | Paint-only; no layout shift; one declared use site (D-14) |
| `color` / `background-color` / `text-decoration-color` | YES (existing hover/focus state transitions only — sub-200ms, MASTER.md §6.2) | Inherited, not a new addition |
| `width`, `height`, `margin`, `padding`, `top/left/right/bottom`, `border-width`, `font-size` | **PROHIBITED** | Layout-triggering; banned to protect Lighthouse Performance ≥99 and CLS ≤0.01 |
| `filter`, `backdrop-filter` | **PROHIBITED** | Expensive paint; not needed for the editorial cadence |

### Duration bands

| Band | Duration | Used by |
|------|----------|---------|
| Snap | 120ms | (legacy hover transitions — already on the floor in MASTER.md §6.2) |
| Tap | 180ms | WorkRow arrow (MOTN-03), chat panel scale-in (MOTN-05) |
| Reveal | 250–350ms (Phase 16 default: **300ms** for section reveals, **250ms** for word-stagger per word) | Scroll-reveal (MOTN-02 / D-06), word-stagger (MOTN-07 / D-10) |
| View-transition | 200ms | Cross-document fade (MOTN-01 / D-21) |
| Loop-fast | 600ms | Typing-dot bounce (MOTN-06 — already live, do not retouch) |
| Loop-slow | 2500ms | Chat bubble pulse (MOTN-04 / D-14) |

**Duration tolerance:** ±20ms is acceptable rounding; outside that band, planner files an amendment to `design-system/MOTION.md`.

### Easing defaults

| Use | Easing | Why |
|-----|--------|-----|
| Entrances (reveal, stagger, scale-in, view-transition) | `ease-out` | Decelerating curve — feels "settling into place" |
| Loops (pulse, typing-dot bounce) | `ease-in-out` | Symmetric breathing rhythm |
| Hover/focus state changes | `ease` (browser default — already in §6.2 transitions) | Inherited |

`cubic-bezier()` custom curves are **prohibited** in Phase 16 — the named easings above cover every spec'd animation. Adding a custom curve later requires a `design-system/MOTION.md` amendment.

### Trigger semantics

- **Astro `astro:page-load`** is the init hook for `motion.ts` (matches existing chat.ts pattern). Cross-document `@view-transition` triggers a full document reload, so re-init is automatic per page navigation.
- **One-shot reveal:** observer calls `unobserve(target)` after first intersect (D-07). Re-entering a page (via browser back/forward) is a fresh observation pass — acceptable, consistent with editorial cadence.
- **Span-wrap idempotency:** `data-stagger-split="true"` attribute set on heading after first wrap; subsequent passes detect and skip (D-13).
- **Pulse pause coordination:** `chat.ts` toggles `data-pulse-paused` attribute on `#chat-bubble` from `openPanel()` / `closePanel()` (replacing the no-op `startPulse`/`stopPulse` stubs at `chat.ts:451-457`). CSS rule: `#chat-bubble[data-pulse-paused] { animation-play-state: paused; }`.

### Reduced-motion contract (MOTN-08)

Per D-23 / D-24 / D-25:

1. Every entrance animation (MOTN-01, MOTN-02, MOTN-05, MOTN-07) lives inside `@media (prefers-reduced-motion: no-preference) { ... }`. With `reduce`, elements render at resting state (opacity: 1, transform: none) immediately — no animation, no stub, no fade.
2. The chat bubble pulse (MOTN-04) is unconditional CSS paired with an explicit `@media (prefers-reduced-motion: reduce) { #chat-bubble { animation: none; } }` neutralizer — the "paired with reduce override" pattern MOTN-08 describes.
3. The WorkRow arrow translateX (MOTN-03) is gated; with `reduce`, the arrow stays opacity 0 (hidden). The accent title underline (color-only state change, sub-200ms — MASTER.md §6.2) still appears as the affordance.
4. `motion.ts` reads `matchMedia('(prefers-reduced-motion: reduce)').matches` before performing word-stagger span-wrap; on `reduce`, no DOM mutation occurs.
5. **First test in every motion feature is the reduced-motion negative case** (ROADMAP "Reduced-Motion Contract" gate).

### Lighthouse gate (MOTN-10)

After motion lands, Lighthouse CI on homepage + one project detail page must hold:

- Performance ≥ 99
- Accessibility ≥ 95
- Best Practices = 100
- SEO = 100
- TBT ≤ 150ms
- CLS ≤ 0.01

**Lighthouse stress conditions to verify:**

- No `will-change` declarations stick (apply only during animation, remove on `animationend` if used at all — researcher recommendation: avoid `will-change` entirely for the listed property whitelist; transform/opacity are GPU-composited without it).
- IntersectionObserver target count stays under 30 per page (D-04 selectors plus 4 scroll-depth sentinels = ~10-20 per project page, well under cap).
- No new `package.json` runtime dep (preferred path).
- No layout-triggering property animates (whitelist enforces this).

### D-26 chat regression gate (applies to MOTN-04, MOTN-05, MOTN-06)

Any change to `chat.ts`, `global.css` chat rules, or `BaseLayout.astro` runs the Phase 7 regression battery before merge:

- XSS sanitization
- Exact-origin CORS
- 5/60s rate limit
- 30s AbortController timeout
- Focus trap re-query on every Tab keypress
- localStorage persistence (50-msg cap / 24h TTL)
- SSE streaming with `async:false` marked parse
- Markdown rendering via DOMPurify strict whitelist
- Clipboard idempotency (JS boolean + DOM data-attribute)

The `data-pulse-paused` attribute toggling in `openPanel`/`closePanel` (D-15) must not race with focus-trap setup; researcher recommendation: toggle the attribute **before** focus-trap activation in `openPanel` and **after** focus restoration in `closePanel`, mirroring the existing `inert` toggle pattern from Phase 12 DEBT-02.
