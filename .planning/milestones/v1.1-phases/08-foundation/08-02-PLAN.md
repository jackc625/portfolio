**Font naming scheme (two-layer, avoids self-reference):** Astro's `<Font cssVariable="--font-display-src" />` emits `:root { --font-display-src: <inlined Geist font stack> }`. Tailwind `@theme { --font-display: var(--font-display-src), system-ui, sans-serif }` then declares the `font-display` utility with a value that references the Astro-set var. The two-layer naming (`--font-*-src` from Astro, `--font-*` from Tailwind `@theme`) is intentional: it makes the distinction visible and prevents the self-reference bug where `:root { --font-display: var(--font-display), ... }` resolves to CSS `initial` because the var references itself.

**Note:** After this plan lands, the build will be DIRTY because downstream files (chat.ts, ChatWidget.astro, components, pages, Header) still reference the old `--token-*` names and `bg-bg-primary` / `text-text-primary` Tailwind classes. That is expected and gets fixed in Plans 03-06. The gate in Plan 08 is what verifies everything is clean. Local `npm run check` mid-wave WILL show errors — the planner accepts this. Do NOT attempt to run the full gate until Plan 06 lands.

The `--color-*: initial` + `--font-*: initial` reset pattern is Tailwind v4 blessed for wiping the default palette. PRESERVE IT. Only replace the bridge body.

The current v1.0 file (lines 123-134) already uses a two-layer naming scheme: Astro sets `--font-heading` / `--font-sans` / `--font-code`, and Tailwind @theme declares `--font-display` / `--font-body` / `--font-mono` that reference those Astro-set vars. This plan keeps the two-layer pattern but renames the Astro side from `--font-heading` / `--font-sans` / `--font-code` to `--font-display-src` / `--font-body-src` / `--font-mono-src` so the relationship is self-evident.

**Why the two-layer pattern is required:** If both sides use the same name (e.g. Astro emits `:root { --font-display: "Geist", ... }` AND Tailwind @theme emits `:root { --font-display: var(--font-display), system-ui, sans-serif }`), the Tailwind declaration references itself and CSS resolves it to `initial`, which breaks the font handoff entirely. The fix is distinct names on each side.

