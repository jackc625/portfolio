Addresses review concern (Gemini MEDIUM + Codex MEDIUM): MDX scoped style bleed mitigated by using `.prose-editorial` as the wrapper class name instead of bare `.body`. The `.prose-editorial` class is specific enough to avoid collisions with any global `.body` class. All `:global()` selectors are scoped under `.prose-editorial` so they only affect MDX-rendered elements inside that wrapper.

Addresses review concern (Codex MEDIUM): External links row handles all absence cases -- the entire `.project-links` div is wrapped in a conditional that checks `project.data.githubUrl || project.data.demoUrl`. When both are absent (which is the case for 5 of 6 projects currently), no empty wrapper div is emitted.

**Frontmatter:**
```typescript
import { getCollection, render, type CollectionEntry } from "astro:content";
import BaseLayout from "../../layouts/BaseLayout.astro";
import Container from "../../components/primitives/Container.astro";
import NextProject from "../../components/NextProject.astro";
import ArticleImage from "../../components/ArticleImage.astro";

export async function getStaticPaths() {
  // IDENTICAL sort as index.astro and projects.astro -- deterministic order
  const allProjects = (await getCollection("projects")).sort(
    (a, b) => a.data.order - b.data.order
  );
  return allProjects.map((project, idx) => ({
    params: { id: project.id },
    props: {
      project,
      // D-10: Next project by order ascending with wrap-around
      nextProject: allProjects[(idx + 1) % allProjects.length],
    },
  }));
}

interface Props {
  project: CollectionEntry<"projects">;
  nextProject: CollectionEntry<"projects">;
}

const { project, nextProject } = Astro.props;
const { Content } = await render(project);
```

Key: NextProject is computed in `getStaticPaths` using order-ascending wrap-around per D-10. Sorting is identical across all 3 pages (homepage, projects index, project detail) for deterministic row ordering. The `render()` function extracts the MDX `Content` component.
