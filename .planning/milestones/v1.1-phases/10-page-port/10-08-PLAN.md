  <action>
    Three UAT gaps in chat.ts — all are surgical single-line or few-line fixes.

    **Fix A — User message bubble border-radius (UAT Test 11, cosmetic):**

    In `createUserMessageEl()` around line 254, the inline CSS sets
    `border-radius: 12px 12px 4px 12px`. Change to `border-radius: 0`.

    In the history replay block around line 545, the inline CSS also sets
    `border-radius: 12px 12px 4px 12px`. Change to `border-radius: 0`.

    Both locations construct user message bubble elements — search for every occurrence of
    `border-radius: 12px 12px 4px 12px` in the file and change to `border-radius: 0`.
    There should be exactly 2 occurrences.

    **Fix B — Message insertion order (UAT Test 12, major):**

    Root cause: When chat history is loaded from localStorage (around line 577), the code uses
    `$messagesArea.appendChild(fragment)` which appends history messages AFTER the typing
    indicator element in the DOM. Then when new messages are sent, they use
    `$messagesArea.insertBefore(el, $typingIndicator)` which places them BEFORE the typing
    indicator — meaning new messages end up above the historical messages.

    Fix: Change line 577 from:
    ```js
    $messagesArea.appendChild(fragment); // Single DOM operation
    ```
    To:
    ```js
    $messagesArea.insertBefore(fragment, $typingIndicator); // Insert before typing indicator so new messages appear after history
    ```

    This ensures history messages are placed before the typing indicator, matching the same
    insertion point that `sendMessage()` uses for new messages. New messages will then
    appear after history (both are before the typing indicator, in DOM order).

    **Fix C — Typing dots animation (UAT Test 12, major):**

    In `startTypingDots()` at lines 441-448, the function explicitly disables CSS animation:
    ```js
    dots.forEach((dot) => {
      dot.style.animation = "none";
      dot.style.opacity = "0.5";
    });
    ```

    The CSS in global.css already defines `@keyframes typing-bounce` and applies it to
    `.typing-dot` via `animation: typing-bounce 600ms var(--dot-delay) ease-in-out infinite`.
    The JS override kills this.

    Fix: Remove the `dot.style.animation = "none"` line and the `dot.style.opacity = "0.5"`
    line. Replace the function body with a no-op comment or simply leave the function body
    empty — the CSS handles the animation automatically when the typing indicator becomes
    visible (`display: flex`).

    Updated function:
    ```ts
    async function startTypingDots(_container: HTMLElement): Promise<void> {
      // CSS @keyframes typing-bounce in global.css handles animation automatically
      // when #chat-typing display changes to flex via showTyping()
    }
    ```

    Note: Prefix the `container` parameter with underscore since it is no longer used,
    to avoid lint errors from no-unused-vars rule.

    Update the comment above the function (line 442) from "Use CSS-based opacity fallback
    (no GSAP bounce — Phase 10 CHAT-02 may restore via @keyframes)" to reflect that the
    CSS keyframes are now active.
  </action>
