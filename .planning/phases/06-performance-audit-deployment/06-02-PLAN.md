**Lighthouse Audit Procedure:**

**1. Build and start preview server:**
```bash
rtk pnpm build
rtk pnpm preview --port 4321 &
# Wait for server to be ready on localhost:4321
sleep 4
```

**2. Run Lighthouse on key pages (mobile mode, all categories):**

Audit at minimum these pages (most important first):
- Home: `rtk npx lighthouse http://localhost:4321/ --output=json --output-path=./lh-home.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- About: `rtk npx lighthouse http://localhost:4321/about/ --output=json --output-path=./lh-about.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- Projects: `rtk npx lighthouse http://localhost:4321/projects/ --output=json --output-path=./lh-projects.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- Resume: `rtk npx lighthouse http://localhost:4321/resume/ --output=json --output-path=./lh-resume.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- Contact: `rtk npx lighthouse http://localhost:4321/contact/ --output=json --output-path=./lh-contact.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- 1 case study page: `rtk npx lighthouse http://localhost:4321/projects/project-alpha/ --output=json --output-path=./lh-casestudy.json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`

Use `--chrome-flags="--headless=new"` (the new headless mode for Chrome). If `rtk npx lighthouse` is not available, install it first: `rtk pnpm add -D lighthouse`.

**3. Parse scores AND metrics from JSON output:**

For each page, extract the four category scores PLUS LCP and CLS:
```bash
node -e "
  const r=require('./lh-home.json');
  const p=r.categories.performance.score*100;
  const a=r.categories.accessibility.score*100;
  const bp=r.categories['best-practices'].score*100;
  const seo=r.categories.seo.score*100;
  const lcp=r.audits['largest-contentful-paint'].numericValue/1000;
  const cls=r.audits['cumulative-layout-shift'].numericValue;
  console.log('P:',p,'A:',a,'BP:',bp,'SEO:',seo,'LCP:',lcp.toFixed(2)+'s','CLS:',cls.toFixed(3));
  if(p<90||a<90||bp<90||seo<90) console.error('FAIL: Score below 90');
  if(lcp>2) console.error('FAIL: LCP '+lcp.toFixed(2)+'s exceeds 2s (PERF-02)');
  if(cls>0.1) console.error('FAIL: CLS '+cls.toFixed(3)+' exceeds 0.1 (PERF-05)');
"
```

**4. If any score is below 90, OR LCP > 2s, OR CLS > 0.1, fix the issues:**

Common fixes to apply if needed:
- **Performance < 90 / LCP > 2s:** Check LCP element, verify GSAP is lazy-loaded, check for render-blocking resources. If TBT is high on home page, consider `requestIdleCallback` for canvas init. If CLS is flagged, check SplitText reflow or font loading.
- **CLS > 0.1:** Check SplitText reflow (lines splitting causes layout shift), check font loading (ensure font-display: swap + fallback metrics are set), check image dimensions (ensure width/height or aspect-ratio set on all images).
- **Accessibility < 90:** Check contrast ratios (especially in light theme), verify all interactive elements have accessible names, check heading hierarchy, verify landmark regions.
- **Best Practices < 90:** Check for deprecated APIs, console errors, missing HTTPS redirects, proper image aspect ratios.
- **SEO < 90:** Check meta descriptions, canonical URLs, robots.txt validity, structured data.

**5. Verify both themes independently (D-09):**

After completing dark-theme Lighthouse audits, verify light theme contrast compliance:

**Approach A (preferred): Run Lighthouse with light theme active.**
Before the audit, inject JavaScript to set the theme via Chrome DevTools Protocol or by modifying the preview page. Alternatively, create a tiny test script:
```bash
node -e "
  // After starting preview server, open a page with light theme forced
  const {execSync} = require('child_process');
  // Run Lighthouse with extra headers or a script that sets localStorage
  execSync('npx lighthouse http://localhost:4321/ --output=json --output-path=./lh-home-light.json --chrome-flags=\"--headless=new\" --screenEmulation.disabled', {stdio:'inherit'});
"
```

If Lighthouse cannot easily switch themes, use **Approach B: Token-value proof.**
Extract the actual color token values from `src/styles/global.css` for both themes and compute contrast ratios:
- Dark theme: text color token (e.g., `--color-text-primary`) vs background color token (e.g., `--color-bg-primary`)
- Light theme (under `[data-theme="light"]`): same token pairs
- Use the WCAG AA contrast formula: (L1 + 0.05) / (L2 + 0.05) >= 4.5 for normal text, >= 3.0 for large text
- Document the specific token values and their computed contrast ratios
- All body text pairs must achieve 4.5:1+, heading/large text pairs must achieve 3.0:1+

If **either approach** reveals contrast failures, fix the failing token values in `src/styles/global.css`.

After each fix:
- Rebuild: `rtk pnpm build`
- Restart preview: kill old server, `rtk pnpm preview --port 4321 &`
- Re-run Lighthouse on the affected page

**Target:** All 4 categories at 90+ on all tested pages, LCP < 2s on all pages, CLS < 0.1 on all pages. Per RESEARCH Pitfall 3, scores can vary 5-10 points between runs. Run twice if a score is borderline (85-92).
