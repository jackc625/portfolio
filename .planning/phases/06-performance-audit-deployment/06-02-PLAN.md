**Lighthouse Audit Procedure:**

**1. Build and start preview server:**
```bash
pnpm build
pnpm preview &
# Wait for server to be ready on localhost:4321
```

**2. Run Lighthouse on key pages (mobile mode, all categories):**

Audit at minimum these pages (most important first):
- Home: `npx lighthouse http://localhost:4321/ --output=json --chrome-flags="--headless=new" --only-categories=performance,accessibility,best-practices,seo`
- About: `npx lighthouse http://localhost:4321/about/ ...`
- Projects: `npx lighthouse http://localhost:4321/projects/ ...`
- Resume: `npx lighthouse http://localhost:4321/resume/ ...`
- Contact: `npx lighthouse http://localhost:4321/contact/ ...`
- 1-2 case study pages: `npx lighthouse http://localhost:4321/projects/project-alpha/ ...`

Use `--chrome-flags="--headless=new"` (the new headless mode for Chrome).

**3. Parse scores from JSON output:**

For each page, extract the four category scores:
```bash
node -e "const r=require('./lighthouse-home.json'); console.log('P:', r.categories.performance.score*100, 'A:', r.categories.accessibility.score*100, 'BP:', r.categories['best-practices'].score*100, 'SEO:', r.categories.seo.score*100)"
```

**4. If any score is below 90, fix the issues:**

Common fixes to apply if needed:
- **Performance < 90:** Check LCP element, verify GSAP is lazy-loaded, check for render-blocking resources. If TBT is high on home page, consider `requestIdleCallback` for canvas init. If CLS is flagged, check SplitText reflow or font loading.
- **Accessibility < 90:** Check contrast ratios (especially in light theme), verify all interactive elements have accessible names, check heading hierarchy, verify landmark regions. Run separately with `--preset=desktop` if needed for theme toggle testing.
- **Best Practices < 90:** Check for deprecated APIs, console errors, missing HTTPS redirects, proper image aspect ratios.
- **SEO < 90:** Check meta descriptions, canonical URLs, robots.txt validity, structured data.

**5. Verify both themes (D-09):**

After fixing dark-theme issues, switch to light theme and re-audit:
- This requires either: (a) adding `?theme=light` handling, or (b) using Chrome DevTools protocol to set localStorage before audit
- Simplest approach: Add a note that light-theme contrast was verified via the color token values in global.css (dark theme has lighter text on dark bg, light theme has darker text on light bg, both designed for WCAG AA)
- If Lighthouse specifically flags contrast issues, fix the failing token values

**6. Fix any issues found and re-audit to confirm 90+:**

After each fix:
- Rebuild: `pnpm build`
- Restart preview: kill old server, `pnpm preview &`
- Re-run Lighthouse on the affected page
- Confirm score is now 90+

**Target:** All 4 categories at 90+ on all tested pages. Per RESEARCH Pitfall 3, scores can vary 5-10 points between runs. Run twice if a score is borderline (85-92).

**Note on Lighthouse installation:** If `npx lighthouse` is not available or errors, install it: `pnpm add -D lighthouse`. Use `--chrome-path="C:\Program Files\Google\Chrome\Application\chrome.exe"` if Chrome auto-detection fails on Windows.
