    **Create `src/scripts/animations.ts`:**

    This is the centralized GSAP animation script. It handles all scroll-triggered reveals, SplitText heading animations, and stagger effects. Uses gsap.context() for View Transitions lifecycle compatibility (per RESEARCH Pattern 2).

    ```typescript
    import gsap from 'gsap';
    import { ScrollTrigger } from 'gsap/ScrollTrigger';
    import { SplitText } from 'gsap/SplitText';

    gsap.registerPlugin(ScrollTrigger, SplitText);

    let ctx: gsap.Context | null = null;

    function initAnimations() {
      // Check reduced motion first (D-13, ANIM-04)
      if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
        // Show all content in final state, no animations
        gsap.set('[data-animate]', { opacity: 1, y: 0 });
        return;
      }

      ctx = gsap.context(() => {
        // 1. Section reveals (D-07, D-08)
        // Every section with data-animate="section" gets a fade-in + slide-up
        gsap.utils.toArray<HTMLElement>('[data-animate="section"]').forEach((el) => {
          gsap.from(el, {
            opacity: 0,
            y: 24,
            duration: 0.8,
            ease: 'power2.out',
            scrollTrigger: {
              trigger: el,
              start: 'top 85%',
              once: true,
            },
          });
        });

        // 2. Stagger items (D-09)
        // Project cards, skill groups, contact channels stagger in sequentially
        gsap.utils.toArray<HTMLElement>('[data-animate-container="stagger"]').forEach((container) => {
          const items = container.querySelectorAll('[data-animate="stagger-item"]');
          if (items.length === 0) return;
          gsap.from(items, {
            opacity: 0,
            y: 24,
            duration: 0.6,
            stagger: 0.1,
            ease: 'power2.out',
            scrollTrigger: {
              trigger: container,
              start: 'top 85%',
              once: true,
            },
          });
        });

        // 3. SplitText display headings (D-11)
        // Line-by-line slide-up reveal on display-sized headings only
        gsap.utils.toArray<HTMLElement>('[data-animate="split-text"]').forEach((el) => {
          SplitText.create(el, {
            type: 'lines',
            mask: 'lines',
            autoSplit: true,
            onSplit(self) {
              return gsap.from(self.lines, {
                y: '100%',
                opacity: 0,
                duration: 0.7,
                stagger: 0.12,
                ease: 'power2.out',
              });
            },
          });
        });
      });
    }

    function cleanupAnimations() {
      ctx?.revert();
      ctx = null;
    }

    document.addEventListener('astro:page-load', initAnimations);
    document.addEventListener('astro:before-preparation', cleanupAnimations);
    ```

    **CRITICAL:** Do NOT set `opacity: 0` in CSS for `[data-animate]` elements. GSAP sets initial state via `gsap.from()`. If GSAP fails to load, content remains visible (RESEARCH Pitfall 7).

  <action>
    Enhance the existing CanvasHero.astro particle flow field with three improvements per D-24 and UI-SPEC Canvas Hero Enhancements section.

    **1. Mouse/cursor influence:**
    Add a `mousemove` event listener on the canvas container. Track cursor position in `mouseX`, `mouseY` variables. In the particle update loop, for each particle:
    - Calculate distance from particle to cursor: `dx = mouseX - p.x`, `dy = mouseY - p.y`, `dist = Math.sqrt(dx*dx + dy*dy)`
    - If `dist < 150` (150px influence radius), compute influence: `influence = (1 - dist / 150) * 0.3` (linear falloff, max 0.3 radians)
    - Add `influence` to the noise-derived angle, biased toward cursor direction: `angle += Math.atan2(dy, dx) * influence`
    - On touch devices (check `matchMedia('(hover: hover)')` or skip if no `mousemove`): no mouse influence for performance.
    - Initialize `mouseX = -1000, mouseY = -1000` (off-screen) so no influence until mouse enters.
    - Add cleanup: `canvas.removeEventListener('mousemove', onMouseMove)` in the cleanup function.

    **2. Parallax depth layers:**
    Modify the `Particle` interface to include a `depth` field:
    ```typescript
    interface Particle {
      x: number; y: number; prevX: number; prevY: number;
      depth: number; // 0.5, 0.75, or 1.0
    }
    ```
    In `initParticles()`, assign each particle a random depth from `[0.5, 0.75, 1.0]` (equal probability).
    In the `draw()` loop:
    - Multiply speed by `p.depth`: `p.x += Math.cos(angle) * speed * p.depth`
    - Set stroke opacity based on depth: instead of fixed `0.18`, use `0.18 * p.depth`
    - Set line width based on depth: `ctx.lineWidth = p.depth === 0.5 ? 0.3 : p.depth === 0.75 ? 0.4 : 0.6`
    - Set strokeStyle per-particle (since opacity varies): `ctx.strokeStyle = \`rgba(${accent.r}, ${accent.g}, ${accent.b}, ${0.18 * p.depth})\``

    Note: Moving strokeStyle inside the loop has a performance cost. To mitigate, group particles by depth and draw each group in a batch (3 passes instead of per-particle style changes). Or simply set strokeStyle per-particle -- at 400-1000 particles, this is fast enough on modern hardware.

    **3. Parameter tuning:**
    - Change `noiseScale` from `0.003` to `0.0025` (smoother, more flowing curves per UI-SPEC)
    - Change `speed` from `1.5` to `1.2` (calmer, more meditative per UI-SPEC)
    - Stroke base opacity is now `0.18` (up from `0.15`) but modified by depth (see above)
    - Line width now varies by depth instead of fixed `0.5` (see above)

    **Preserve existing patterns:**
    - Keep `prefers-reduced-motion` check — `drawStaticFrame()` for reduced motion users
    - Keep View Transitions lifecycle (`astro:page-load`, `astro:before-preparation`)
    - Keep the `theme-changed` listener added in Plan 01
    - Keep `isMobile` particle count adjustment (400 vs 1000)
    - Update `drawStaticFrame` to also use depth-varied opacity for the static dots
  </action>
