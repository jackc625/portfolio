**2. Replace the broken influence formula (lines 131-138):**
Replace the entire mouse influence block:
```javascript
// Mouse influence (D-24): particles near cursor are gently pushed
const dx = mouseX - p.x;
const dy = mouseY - p.y;
const dist = Math.sqrt(dx * dx + dy * dy);
if (dist < mouseRadius) {
  const influence = (1 - dist / mouseRadius) * mouseInfluenceMax;

}
```
With the corrected angular-difference formula:
```javascript
// Mouse influence (D-24): particles near cursor bend toward mouse direction
const dx = mouseX - p.x;
const dy = mouseY - p.y;
const dist = Math.sqrt(dx * dx + dy * dy);
if (dist < mouseRadius) {
  const influence = (1 - dist / mouseRadius) * mouseInfluenceMax;
  const mouseAngle = Math.atan2(dy, dx);
  let angleDiff = mouseAngle - angle;
  // Normalize to [-PI, PI] for shortest-path rotation
  while (angleDiff > Math.PI) angleDiff -= 2 * Math.PI;
  while (angleDiff < -Math.PI) angleDiff += 2 * Math.PI;
  angle += angleDiff * influence;
}
```

**Why this fixes the bug:** The old formula multiplied raw atan2 (direction to mouse as an absolute angle) by influence and added it. When mouse was directly right of a particle, atan2 returned ~0, so the adjustment was zero regardless of proximity. The new formula computes the *difference* between the particle's current flow heading and the mouse direction, normalized to the shortest rotation path, then applies a fraction of that difference. This produces directionally consistent influence — particles always rotate toward the mouse proportional to proximity, regardless of the mouse's angle.

**Do NOT change anything else** — the event listener fix from 05-05 (section-level mousemove) is correct and must be preserved. The draw loop structure, trail rendering, depth system, and all other code remain unchanged.
