   **Focus trap cleanup:**
   - On `astro:before-preparation`, remove keydown listener if panel is open to prevent stale handlers
   - On `astro:page-load`, re-attach if panel state indicates it should be open (transition:persist preserves DOM but not JS state)

   **Implementation approach:** Add a `setupFocusTrap(panel: HTMLElement)` function that attaches the keydown handler. Call it when panel opens. Remove handler when panel closes. The function should:
   ```typescript
   function setupFocusTrap(panel: HTMLElement): () => void {
     const handler = (e: KeyboardEvent) => {
       if (e.key === "Escape") {
         closePanel();
         return;
       }
       if (e.key !== "Tab") return;
       const focusable = panel.querySelectorAll<HTMLElement>(
         'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
       );
       if (focusable.length === 0) return;
       const first = focusable[0];
       const last = focusable[focusable.length - 1];
       if (e.shiftKey && document.activeElement === first) {
         e.preventDefault();
         last.focus();
       } else if (!e.shiftKey && document.activeElement === last) {
         e.preventDefault();
         first.focus();
       }
     };
     panel.addEventListener("keydown", handler);
     return () => panel.removeEventListener("keydown", handler);
   }
   ```
   Store the cleanup function and call it when panel closes.

   function trackChatEvent(action: string, label?: string): void {
     document.dispatchEvent(new CustomEvent("chat:analytics", {

     }));
