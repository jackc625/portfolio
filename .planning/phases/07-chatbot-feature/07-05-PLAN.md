  <what-built>
Complete AI chatbot feature including:
- Floating chat bubble with pulse animation in bottom-right corner
- Expandable 400x500px chat panel with scale+fade GSAP animation
- Full chat UI: header ("Ask Jack's AI"), starter chips, message area, typing indicator, input with character count, privacy note
- Streaming responses from Claude Haiku via SSE
- Markdown rendering with XSS sanitization (bold, links, lists)
- Copy-to-clipboard on bot messages
- Focus trapping and full keyboard accessibility
- Mobile full-screen takeover at <768px
- Dark/light theme awareness using design system tokens
- Chat persists across page navigation via transition:persist
- Nudge message after ~15 messages
- Error handling with friendly inline messages
  </what-built>
  <how-to-verify>
**Prerequisites:** Ensure `.dev.vars` file exists at project root with a valid `ANTHROPIC_API_KEY`. Start dev server: `npm run dev`

**Desktop verification (visit http://localhost:4321):**
1. Chat bubble is visible in bottom-right corner with pulse animation
2. Click bubble — panel opens with smooth scale+fade animation
3. Panel shows "Ask Jack's AI" header, 4 starter chips, input field, "Conversations are not stored." note
4. Click a starter chip — it sends the question and shows typing indicator, then streaming bot response
5. Type a message and press Enter — message appears right-aligned, bot responds with streaming text
6. Bot response renders markdown (bold, links, bullet lists) correctly
7. Hover over a bot message — copy button appears. Click it — icon changes to checkmark briefly
8. Type a long message — character count appears as "X/500", changes color near limit
9. Press Shift+Enter — adds a newline in the textarea
10. Press Escape — panel closes, focus returns to bubble
11. Click bubble again — panel reopens with previous conversation intact
12. Navigate to a different page — chat widget and conversation persist
13. Toggle dark/light theme — chat widget colors update to match

**Mobile verification (browser DevTools responsive mode at <768px):**
14. Click bubble — panel opens as full-screen overlay (no border-radius, fills viewport)
15. Input area respects safe area at bottom
16. Close button works to dismiss full-screen panel

**Accessibility verification:**
17. Tab through the panel — focus cycles between interactive elements (close button, starter chips or messages, textarea, send button)
18. Focus does not leave the panel while it is open

**Error handling (optional — requires breaking the API temporarily):**
19. If API key is invalid or removed, sending a message shows "Sorry, I'm having trouble right now. Try again in a moment."
