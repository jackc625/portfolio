## Spacing Scale

Declared values (must be multiples of 4). Existing tokens from `global.css`:

| Token | Value | Usage in Chat Widget |
|-------|-------|----------------------|
| xs | 4px (`--token-space-xs`) | Icon gaps, inline padding between send icon and input edge |
| sm | 8px (`--token-space-sm`) | Message bubble internal inline padding, gap between avatar and message, gap between starter chips |
| md | 16px (`--token-space-md`) | Panel body padding, gap between messages, input area padding |
| lg | 24px (`--token-space-lg`) | Panel header padding, gap between message groups |
| xl | 32px (`--token-space-xl`) | Bottom offset of floating bubble from viewport edge |
| 2xl | 48px (`--token-space-2xl`) | Not used in this phase |
| 3xl | 64px (`--token-space-3xl`) | Not used in this phase |

Exceptions:
- Floating bubble button: 48px x 48px (touch target, meets 44px minimum per WCAG 2.5.5)
- Right/bottom offset of bubble from viewport edge: 24px (`--token-space-lg`)
- Chat panel width: 400px (D-02, fixed value -- not a spacing token)
- Chat panel height: 500px (D-02, fixed value -- not a spacing token)
- Mobile: full viewport width and height (D-05)

Source: global.css `:root` spacing tokens, 07-CONTEXT.md D-02/D-05

## Typography

Uses existing fluid type scale from `global.css`. All text in the chat widget uses `font-body` (Inter) unless otherwise specified.

| Role | Token / Size | Weight | Line Height | Usage in Chat Widget |
|------|-------------|--------|-------------|----------------------|
| Body | `--token-text-base` (clamp 15px-17px) | 400 (regular) | 1.5 | Message text (user and bot), input field text |
| Label | `--token-text-sm` (clamp 12px-14px) | 500 (medium) | 1.4 | Header label "Ask Jack's AI", privacy note, character count, timestamp, starter chip text |
| Heading | `--token-text-heading` (clamp 20px-32px) | 600 (semibold) | 1.2 | Not used in chat widget (no headings needed) |
| Display | `--token-text-display` (clamp 40px-88px) | 700 (bold) | 1.1 | Not used in chat widget |

Additional typography rules:
- Bot markdown content: `font-body` at `--token-text-base`, weight 400, line-height 1.5
- Bot markdown bold: weight 600
- Bot markdown code spans: `font-mono` (IBM Plex Mono) at `--token-text-sm`, weight 400
- Bot markdown links: `color-accent`, underline on hover, weight 400
- Bot markdown bullet lists: `--token-space-sm` left indent per level, `--token-space-xs` gap between items
- Character count near limit (>450): weight 500, `color-warning`
- Character count at limit (500): weight 500, `color-destructive`

Source: global.css `:root` typography tokens, astro.config.mjs font configuration, 07-CONTEXT.md D-04/D-38

**Accent reserved for** (explicit list -- never "all interactive elements"):
1. Send button icon color
2. Floating bubble icon color (the chat icon itself)
3. Floating bubble pulse animation glow
4. Bot message markdown links
5. Copy button icon on hover
6. Starter chip text on hover
7. Focus ring on all interactive elements (established codebase pattern: `focus-visible:ring-accent`)

**Bot message bubble color:** `--token-bg-primary` (same as panel background -- bot messages are visually distinguished by left-alignment and a subtle left border of `--token-border` at 2px width, not by background color)

## Component Inventory

All components are `.astro` files for static markup or vanilla JS `<script>` tags for interactivity (established pattern from D-04, code_context). No React/Vue islands.

### ChatWidget.astro (parent container)
- Injected in `BaseLayout.astro` before `</body>`, after Footer
- Uses `transition:persist` to survive page navigation (D-07)
- Contains: ChatBubble + ChatPanel
- Manages open/close state via vanilla JS

### ChatBubble (within ChatWidget)
- Fixed position: bottom 24px, right 24px (D-01, D-03)
- Size: 48px x 48px circle
- Background: `--token-accent`
- Icon: inline SVG message bubble, 24px, color white (high contrast on accent bg)
- Subtle pulse animation: box-shadow glow using `--token-accent` at 40% opacity, 2s infinite ease-in-out (D-03)
- Hover: scale(1.05), `--token-accent-hover` background
- `aria-label="Open chat"` / `aria-label="Close chat"` (toggles with state)
- `aria-expanded="true/false"`
- z-index: 50

### ChatPanel (within ChatWidget)
- Fixed position: bottom 80px (bubble height + gap), right 24px
- Width: 400px, Height: 500px (D-02)
- Mobile: full viewport width, full viewport height minus safe area (D-05)
- Background: `--token-bg-primary`
- Border: 1px solid `--token-border`
- Border radius: 12px (desktop), 0px (mobile full-screen)
- Box shadow: `0 8px 32px rgba(0,0,0,0.25)` (dark), `0 8px 32px rgba(0,0,0,0.1)` (light)
- z-index: 49
- Open animation: GSAP scale(0.9) + opacity(0) to scale(1) + opacity(1), 300ms, ease power2.out, transform origin bottom-right (D-06)
- Close animation: reverse of open, 200ms
- Structure: Header + MessageArea + InputArea (flex column, full height)

### ChatHeader (within ChatPanel)
- Height: 48px
- Background: `--token-bg-secondary`
- Border bottom: 1px solid `--token-border`
- Padding: 0 16px (`--token-space-md`)
- Layout: flex row, justify-between, align-center
- Left: label "Ask Jack's AI" in `--token-text-sm` at weight 500, `--token-text-primary` color
- Right: close button (X icon, 20px, `--token-text-muted`, hover `--token-text-primary`)
- Close button: `aria-label="Close chat"`, Escape key also closes (D-33)

### MessageArea (within ChatPanel)
- Flex: 1 (fills available space between header and input)
- Overflow-y: auto, smooth scroll
- Padding: 16px (`--token-space-md`)
- `aria-live="polite"` for screen reader announcements (D-32)
- `role="log"` for assistive technology
- Auto-scrolls to bottom on new message

### StarterChips (within MessageArea, shown when empty)
- Displayed when conversation has 0 messages
- 3-4 clickable pill buttons (D-27)
- Layout: flex wrap, gap 8px (`--token-space-sm`)
- Each chip: `--token-bg-secondary` background, 1px solid `--token-border`, border-radius 9999px (full pill)
- Padding: 6px 12px
- Text: `--token-text-sm`, weight 400, `--token-text-secondary`
- Hover: border `--token-border-hover`, text `--token-accent`
- `role="button"`, `tabindex="0"`

### MessageBubble (within MessageArea)
- **User messages:** right-aligned, background `--token-bg-secondary`, border-radius 12px 12px 4px 12px, max-width 85%
- **Bot messages:** left-aligned, background transparent, left border 2px solid `--token-border`, border-radius 0px, padding-left 12px, max-width 90%
- Message padding: 10px 14px (user), 8px 0 8px 12px (bot -- left border provides visual boundary)
- Message gap: 8px (`--token-space-sm`) between consecutive same-sender messages, 16px (`--token-space-md`) between sender switches
- Bot message text: `--token-text-secondary` (slightly muted to differentiate from user text)
- User message text: `--token-text-primary`

### TypingIndicator (within MessageArea)
- Shown after user sends message, before first streaming token (D-28)
- Three dots, 6px circles, `--token-text-muted` color
- Bounce animation: staggered 0.15s delay per dot, 0.6s infinite ease-in-out, translateY(-4px)
- Appears in bot message position (left-aligned, with left border)

### CopyButton (within each bot MessageBubble)
- Appears on hover (desktop) or always visible (mobile/touch) (D-30)
- Position: top-right of bot message, offset -8px up from message top
- Size: 28px x 28px
- Icon: clipboard SVG, 14px, `--token-text-muted`
- Hover: icon color `--token-accent`
- After copy: icon changes to checkmark for 2 seconds, color `--token-success`
- `aria-label="Copy message"`
- Tooltip not needed -- icon is self-explanatory with aria-label

### InputArea (within ChatPanel)
- Border top: 1px solid `--token-border`
- Padding: 12px 16px (`--token-space-md`)
- Background: `--token-bg-primary`
- Layout: flex row, gap 8px (`--token-space-sm`), align-end
- **Text input:** `<textarea>` with auto-grow (1 row min, 4 rows max), background `--token-bg-secondary`, border 1px solid `--token-border`, border-radius 8px, padding 10px 12px, placeholder "Ask me anything...", `--token-text-muted` placeholder color
- Focus: border-color `--token-border-hover`, ring 2px `--token-accent` (consistent with codebase focus pattern)
- **Send button:** 36px x 36px circle, background `--token-accent`, icon arrow-up SVG 18px white, disabled state opacity 0.4, `aria-label="Send message"`
- **Character count:** below textarea, right-aligned, `--token-text-sm`, `--token-text-muted` by default
- Input disabled state while bot streaming (D-34): textarea and send button both opacity 0.5, pointer-events none

### PrivacyNote (within ChatPanel)
- Positioned at bottom of panel, below input area
- Text: "Conversations are not stored." (D-37)
- Style: `--token-text-sm`, `--token-text-muted`, centered, padding 4px 0 8px 0

### NudgeMessage (system message)
- Appears after ~15 messages (D-29)
- Styled as a centered system message, not a bot bubble
- Text: `--token-text-sm`, `--token-text-muted`, italic
- No left border (distinct from bot messages)
- Padding: 8px 16px
- Background: none

## Interaction States

### Floating Bubble
| State | Visual |
|-------|--------|
| Default | `--token-accent` bg, pulse glow animation, scale(1) |
| Hover | `--token-accent-hover` bg, scale(1.05), pulse pauses |
| Active/pressed | scale(0.95), 100ms |
| Focus visible | 2px ring `--token-accent`, ring-offset 2px `--token-bg-primary` |
| Chat open | Bubble rotates icon to X (close), pulse stops, `aria-expanded="true"` |

### Chat Panel
| State | Visual |
|-------|--------|
| Opening | GSAP: scale 0.9 + opacity 0 -> scale 1 + opacity 1, 300ms power2.out from bottom-right origin |
| Open | Fully visible, focus trapped inside (D-33) |
| Closing | GSAP: reverse of opening, 200ms |
| Closed | display:none after animation completes |

### Input Field
| State | Visual |
|-------|--------|
| Empty | Placeholder visible, send button disabled (opacity 0.4) |
| Typing | Placeholder hidden, send button enabled (full opacity), character count appears |
| Characters 1-450 | Count in `--token-text-muted` |
| Characters 451-499 | Count in `--token-warning`, weight 500 |
| Characters 500 (limit) | Count in `--token-destructive`, weight 500, textarea prevents further input |
| Streaming (disabled) | textarea + send button opacity 0.5, pointer-events none |

### Messages
| State | Visual |
|-------|--------|
| User message sent | Appears immediately, right-aligned |
| Bot streaming | Typing indicator shown, then replaced by streaming text as tokens arrive |
| Bot complete | Full message visible, copy button appears on hover |
| Copy button clicked | Icon changes clipboard -> checkmark, `--token-success` color, reverts after 2s |

### Error States
| State | Visual |
|-------|--------|
| API error | Inline bot message: "Sorry, I'm having trouble right now. Try again in a moment." in `--token-text-muted`, italic (D-12) |
| Rate limited | Inline bot message: "You've sent a lot of messages. Please wait a moment before trying again." in `--token-text-muted`, italic |
| Network offline | Inline bot message: "It looks like you're offline. Check your connection and try again." in `--token-text-muted`, italic |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Screen reader announcements | `aria-live="polite"` on message area container (D-32) |
| Message area semantics | `role="log"` on message area |
| Keyboard: Escape | Closes panel, returns focus to bubble (D-33) |

| Keyboard: Enter | Sends message (D-34) |

| Touch targets | Bubble 48px, send button 36px, close button min 44px hit area, starter chips min 44px height |

## Animation Contract

All animations use GSAP (established pattern) and respect `prefers-reduced-motion: reduce`.

| Animation | Trigger | Duration | Easing | Details |
|-----------|---------|----------|--------|---------|
| Bubble pulse | Continuous when panel closed | 2s infinite | ease-in-out | box-shadow: 0 0 0 0px -> 0 0 0 8px `--token-accent` at 40% opacity |
| Bubble hover scale | Mouse enter/leave | 200ms | power2.out | scale(1) -> scale(1.05) |
| Bubble press | Click down/up | 100ms | power2.out | scale(1) -> scale(0.95) |
| Panel open | Bubble click (open) | 300ms | power2.out | scale(0.9) opacity(0) -> scale(1) opacity(1), transform-origin bottom-right |
| Panel close | Bubble click (close) / Escape | 200ms | power2.in | Reverse of open |
| Typing dots | Bot thinking | 600ms infinite staggered | ease-in-out | translateY(0) -> translateY(-4px), 150ms stagger per dot |
| Message appear | New message added | 200ms | power2.out | opacity(0) translateY(8px) -> opacity(1) translateY(0) |
| Copy feedback | Copy button click | Instant swap, revert after 2s | n/a | Clipboard icon -> checkmark icon, color `--token-text-muted` -> `--token-success` |

## Markdown Rendering Contract

Bot responses render markdown (D-38). Library choice is at Claude's Discretion per context doc.

**Recommended library:** `marked` (lightweight, fast, no dependencies, SSR-friendly) + `DOMPurify` (XSS sanitization per D-21, D-25).

**Supported markdown features (whitelist):**
- Bold (`**text**`)
- Italic (`*text*`)
- Inline code (`` `code` ``)
- Links (`[text](url)`) -- open in new tab with `rel="noopener noreferrer"`
- Unordered lists (`- item`)
- Paragraphs (double newline)

**Blocked markdown features:**
- Images (no `<img>` tags from bot output)
- HTML tags (stripped by DOMPurify)
- Script injection (stripped by DOMPurify)
- Headings (bot responses should not contain headings -- overly complex for chat)

**Rendering pipeline:**
1. Raw bot text -> `marked.parse()` with options `{ breaks: true, gfm: true }`
2. Parsed HTML -> `DOMPurify.sanitize()` with allowed tags: `b, strong, em, i, code, a, ul, li, p, br`
3. Sanitized HTML -> inject into message bubble via `innerHTML`

Source: 07-CONTEXT.md D-21, D-25, D-38. Library choice exercised under Claude's Discretion.
