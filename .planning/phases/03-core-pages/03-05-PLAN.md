**DESIGN PRINCIPLES extracted from inspiration sites (apply these):**

1. **Asymmetric spatial composition** (from shiyunlu.com, aither.co): Content does NOT live in a single centered max-w container. Use CSS Grid with named areas or explicit column placements that create intentional asymmetry. Example: hero text pinned to left 60% of viewport while right 40% is breathing space, or content that breaks out of expected alignment.

2. **Typography AS the design** (from artemshcherban.com, bettinasosa.com): The name/headline is not just large text in a container — it IS the visual composition. Use viewport-relative sizing (clamp with vw), negative letter-spacing, line-height below 1.0, and strategic line breaks to create visual tension. Text can overlap, stack asymmetrically, or extend beyond expected boundaries.

3. **Full-bleed sections with varied horizontal rhythms** (from wam.global, aither.co): Different sections use different horizontal margins/padding. The hero might use px-6 md:px-12 lg:px-20 while the projects section uses edge-to-edge treatment, and the about teaser uses a narrow asymmetric column. This variation creates visual rhythm.

4. **Projects woven into typographic flow** (per D-03): Projects are NOT a separate "grid section." They are part of the page's typographic composition — large project titles that are themselves design elements, positioned with intentional spacing and hierarchy. Think: project names at display scale with metadata as subtle annotations.

5. **Minimal but intentional decorative elements**: No film grain overlays, no ghost numbers, no grid line decorations. If a decorative element exists, it must serve a compositional purpose (e.g., a horizontal rule that creates visual rhythm, a vertical line that guides the eye).

**LAYOUT CONCEPT TO IMPLEMENT:**

Create a layout that uses a CSS Grid with asymmetric columns at the page level. The page should NOT follow a symmetric centered-container pattern. Instead:

**Hero area:**
- Use `display: grid` with `grid-template-columns: 1fr minmax(0, 65ch) minmax(2rem, 1fr)` or similar asymmetric grid
- Name positioned with extreme typographic scale — use `font-size: clamp(4rem, 3rem + 10vw, 14rem)` with `line-height: 0.85` and `letter-spacing: -0.05em`
- The name should feel like it owns the entire viewport width — not contained in a box
- Role line ("Software Engineer") as a small mono label, positioned AWAY from the name (e.g., right-aligned while name is left, or placed in a different grid cell) to create spatial tension
- Tagline as body text, positioned with deliberate offset from the name — not directly below in the same column
- CTA positioned independently (not stacked below tagline) — placed in its own grid area

**Projects area:**
- Each project title at `font-size: clamp(1.5rem, 1rem + 3vw, 3.5rem)` in font-display
- Projects as a typographic list where each entry spans different horizontal positions
- Use `border-top` (not border-bottom) for visual separation
- Keep hover: title to text-accent, maintain the link wrapping `<a href="/projects/{slug}">`
- Tech stack as small mono text, positioned as margin annotation (not inline below)

**About teaser:**
- Not a separate "About" section with a heading
- A single pull-quote statement in display typography that reads as editorial — placed in a narrow column offset from center
- The "Read more" link as a subtle mono-text element, not a prominent CTA

**Quick links:**
- "Resume" and "Get in Touch" as minimal text links — can be part of the page's typographic grid, not a boxed section
- Style: font-display at heading scale, text links with hover to accent
