# Phase 3 -- UI Design Contract

> Visual and interaction contract for the four core pages: Home, About, Resume, and Contact. Generated by gsd-ui-researcher.
>
> All design tokens (colors, typography, spacing) and shell components (Header, Footer, MobileMenu) are inherited from Phase 1 and Phase 2 UI-SPECs. This contract specifies page-level composition, layout patterns, component definitions, and copywriting for Phase 3.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro 6 + Tailwind CSS v4 -- not React, shadcn not applicable) |
| Preset | not applicable |
| Component library | none (vanilla Astro components with Tailwind utilities) |

## Spacing Scale

Inherited from Phase 1 UI-SPEC without modification. Phase 3 usage:

| Token | Value | Phase 3 Usage |
|-------|-------|---------------|
| xs | 4px (0.25rem) | Tech stack tag padding, inline icon gaps |
| sm | 8px (0.5rem) | Tag gaps, compact label spacing, skill item padding |
| md | 16px (1rem) | Paragraph spacing, list item gaps, mobile horizontal padding |
| lg | 24px (1.5rem) | Content block gaps, card padding, desktop horizontal padding |
| xl | 32px (2rem) | Major content block separation within a section |
| 2xl | 48px (3rem) | Between-section spacing within a page |
| 3xl | 64px (4rem) | Hero to first content section gap |
| section | clamp(4rem, 3rem + 5vw, 8rem) | Top/bottom padding for each major page section |

### Phase 3 Exceptions

- **Home hero vertical padding:** Uses `min-h-[calc(100vh-4rem)]` (full viewport minus header height) rather than fixed spacing. The hero fills the viewport.
- **Content max-width:** Prose content blocks use `max-w-2xl` (672px) for optimal reading line length (~65-75 characters). Layout containers use `max-w-5xl` (1024px) matching the shell.
- **Featured project items:** Gap between featured project entries is `xl` (32px) on mobile, `2xl` (48px) on desktop.

## Typography

Inherited from Phase 1 UI-SPEC. Phase 3 usage mapping:

| Role | Token | Font | Weight | Line Height | Phase 3 Usage |
|------|-------|------|--------|-------------|---------------|
| Label / Caption | `--token-text-sm` (13-14px fluid) | Sans / Mono | 400 | 1.5 | Tech stack tags (mono), skill category labels (mono), resume section labels, availability badge, contact channel labels |
| Body | `--token-text-base` (15-16px fluid) | Sans | 400 | 1.6 | Paragraph text on About page, resume highlights, contact page description, hero tagline |
| Heading | `--token-text-heading` (20-32px fluid) | Sans | 600 | 1.25 | Page section headings (H2), skill group titles, resume section titles |
| Display | `--token-text-display` (36-80px fluid) | Serif | 400 | 1.05 | Home hero headline, About page headline, Contact page headline |

### Phase 3 Typography Rules

1. **Home hero name:** `--token-text-display` at full scale. Instrument Serif, weight 400. The name "Jack Cutrara" is the largest text element on the entire site. Negative letter-spacing (-0.03em) per Phase 1 spec.
2. **Home hero role line:** `--token-text-heading` in Instrument Sans, weight 400 (not 600). Uses `text-text-secondary` to create visual hierarchy beneath the name. Uppercase with `letter-spacing: 0.1em` for editorial feel.
3. **Home hero tagline:** `--token-text-base` in Instrument Sans, weight 400. `text-text-secondary`. Max 2 lines. This is the "brief intro" per HOME-01.
4. **Featured project titles:** `--token-text-heading` in Instrument Serif, weight 400. These are typographic elements woven into the home design per D-03, not card headings.
5. **About page body text:** `--token-text-base` at 1.6 line-height. First-person tone per D-05. Paragraphs separated by `md` (16px) spacing.
6. **Skills labels:** JetBrains Mono at `--token-text-sm`, weight 400, `text-text-secondary`. The mono treatment signals technical content.
7. **Resume section titles:** `--token-text-heading` in Sans, weight 600. These create scannable structure in the styled summary.
8. **Contact headline:** `--token-text-display` in Serif. The contact page gets the same editorial headline treatment as Home and About per D-07/D-12.

## Color

Inherited from Phase 1 UI-SPEC. Phase 3 specific usage:

| Role | Token | Phase 3 Application |
|------|-------|--------------------|
| Dominant (60%) | `--token-bg-primary` | Page backgrounds for all four pages, hero background |
| Secondary (30%) | `--token-bg-secondary` | Featured project hover areas, skill group cards, resume content card, contact channel cards |
| Accent (10%) | `--token-accent` | Primary CTA button, featured project hover indicator, inline links in prose, availability badge dot |
| Accent hover | `--token-accent-hover` | CTA button hover state, featured project title hover |

### Phase 3 Accent Usage (strict)

Within this phase, accent color appears ONLY on:

1. **"View Projects" CTA button** on home hero -- text color on a transparent/border button, or background on a filled button (see component spec below)
2. **Featured project hover state** -- project title shifts to accent on hover
3. **Inline links** within About page prose text
4. **Availability status dot** on Contact page -- small pulsing dot indicating "open to opportunities"
5. **"Download Resume" button** -- same button treatment as primary CTA
6. **Focus rings** -- inherited from Phase 2

Accent is NOT used on: headings, section backgrounds, decorative borders, skill tags, or static text elements.

### Additional Color Rules

- **Skill tags:** Use `--token-border` for border, `--token-bg-secondary` for subtle background tint. No accent color.
- **Resume content area:** `--token-bg-secondary` background to distinguish the styled resume summary from the page background.
- **Contact channel cards:** `--token-bg-secondary` background with `--token-border` border. Hover state shifts border to `--token-border-hover`.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | One `<h1>` per page. Heading hierarchy: H1 > H2 > H3 (no skipped levels). Sections use `aria-label`. |
| Landmark regions | Each page section wrapped in `<section>` with descriptive `aria-label`. |
| Link purpose | All links have descriptive text or `aria-label`. "View Projects" not "Click here". |
| External links | LinkedIn and GitHub links on Contact page use `target="_blank"` with visually hidden " (opens in new tab)" text appended for screen readers. |
| Download link | Resume PDF button uses `aria-label="Download resume as PDF"`. |
| Availability badge | Dot is decorative (`aria-hidden="true"`). Status text carries the semantic meaning. |
| Reduced motion | Availability pulse animation respects `prefers-reduced-motion: reduce`. |
| Focus indicators | All interactive elements use `focus-visible:ring-2 focus-visible:ring-accent` per Phase 2. |
| Color contrast | All text/background pairs verified against Phase 1 contrast table. `text-text-secondary` on `bg-bg-secondary` requires verification (0.62 L on 0.18 L -- approximately 4.2:1, borderline AA for body text). Use `text-text-primary` for any body-sized text on `bg-bg-secondary` backgrounds. Reserve `text-text-secondary` for large text or labels only on secondary backgrounds. |
