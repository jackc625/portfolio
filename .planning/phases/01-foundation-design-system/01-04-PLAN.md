<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Create GitHub repo and connect Cloudflare Pages</name>
  <files>none -- external service configuration</files>
  <action>
    The user must complete these steps manually because the gh CLI is not installed and Cloudflare Pages dashboard requires browser authentication. Claude cannot automate GitHub repo creation or Cloudflare Pages project setup without these tools.

    **Step 1: Create GitHub repository**

    1. Go to https://github.com/new
    2. Repository name: `portfolio` (or preferred name)
    3. Visibility: **Public** (per D-13 -- source code visible to hiring managers)
    4. Do NOT initialize with README, .gitignore, or license (repo already has content)
    5. Click "Create repository"
    6. Copy the remote URL (HTTPS or SSH) and run locally:
       ```bash
       git remote add origin https://github.com/YOUR_USERNAME/portfolio.git
       git branch -M main
       git push -u origin main
       ```

    **Step 2: Connect Cloudflare Pages**

    1. Go to https://dash.cloudflare.com/ and sign in (create account if needed)
    2. Navigate to "Workers & Pages" in the sidebar
    3. Click "Create" > "Pages" > "Connect to Git"
    4. Authorize Cloudflare to access your GitHub account
    5. Select the `portfolio` repository
    6. Configure build settings:
       - **Production branch:** `main`
       - **Framework preset:** Astro
       - **Build command:** `pnpm run build`
       - **Build output directory:** `dist`
    7. Add environment variable: `NODE_VERSION` = `22`
    8. Click "Save and Deploy"
    9. Wait for first deploy to complete (1-3 minutes)
    10. Note the `*.pages.dev` URL (e.g., `jack-cutrara-portfolio.pages.dev`)

    **Step 3: Verify HTTPS**

    1. Visit the `*.pages.dev` URL in browser
    2. Confirm HTTPS lock icon in address bar
    3. Confirm blank dark page renders (near-black background)

    **Step 4 (optional, can defer): Custom domain**

    Per D-11, custom domain is available at an external registrar. This can be configured now or later:
    1. In Cloudflare Pages project settings, go to "Custom domains"
    2. Add your domain (e.g., `jackcutrara.com`)
    3. Follow DNS instructions to add CNAME record at your registrar pointing to `jack-cutrara-portfolio.pages.dev`
    4. Wait for DNS propagation and SSL certificate (can take up to 24 hours)
  </action>
