    **Layer 1: Raw design tokens on :root (dark theme default)**

    All values copied verbatim from UI-SPEC.md tables:

    ```css
    :root {
      /* === COLOR TOKENS === */
      /* Core Palette (60-30-10 rule) */
      --token-bg-primary: oklch(0.13 0.005 260);
      --token-bg-secondary: oklch(0.18 0.005 260);
      --token-accent: oklch(0.72 0.12 178);
      --token-accent-hover: oklch(0.78 0.14 178);

      /* Text Colors */
      --token-text-primary: oklch(0.93 0.005 260);
      --token-text-secondary: oklch(0.62 0.01 260);
      --token-text-muted: oklch(0.45 0.01 260);

      /* Border & Divider */
      --token-border: oklch(0.25 0.005 260);
      --token-border-hover: oklch(0.35 0.005 260);

      /* Semantic Colors */
      --token-destructive: oklch(0.63 0.2 25);
      --token-success: oklch(0.72 0.15 155);
      --token-warning: oklch(0.78 0.15 85);

      /* === TYPOGRAPHY TOKENS === */
      /* Fluid type scale per UI-SPEC Type Scale table */
      --token-text-sm: clamp(0.8125rem, 0.75rem + 0.3vw, 0.875rem);
      --token-text-base: clamp(0.9375rem, 0.875rem + 0.3vw, 1rem);
      --token-text-heading: clamp(1.25rem, 1rem + 1.25vw, 2rem);
      --token-text-display: clamp(2.25rem, 1.5rem + 3.75vw, 5rem);

      /* === SPACING TOKENS === */
      --token-space-xs: 0.25rem;
      --token-space-sm: 0.5rem;
      --token-space-md: 1rem;
      --token-space-lg: 1.5rem;
      --token-space-xl: 2rem;
      --token-space-2xl: 3rem;
      --token-space-3xl: 4rem;
      --token-space-section: clamp(4rem, 3rem + 5vw, 8rem);
    }
    ```

    **Layer 2: Tailwind @theme bridge**

    After the `:root` block, add the Tailwind import and @theme directive:

    ```css
    @import "tailwindcss";

    @theme {
      --color-*: initial;
      --font-*: initial;

      /* Color tokens -> Tailwind utilities */
      --color-bg-primary: var(--token-bg-primary);
      --color-bg-secondary: var(--token-bg-secondary);
      --color-text-primary: var(--token-text-primary);
      --color-text-secondary: var(--token-text-secondary);
      --color-text-muted: var(--token-text-muted);
      --color-accent: var(--token-accent);
      --color-accent-hover: var(--token-accent-hover);
      --color-border: var(--token-border);
      --color-border-hover: var(--token-border-hover);
      --color-destructive: var(--token-destructive);
      --color-success: var(--token-success);
      --color-warning: var(--token-warning);

      /* Font family tokens -> Tailwind utilities */
      /* Note: these reference the CSS vars set by Astro Fonts API <Font /> component */
      --font-display: var(--font-heading), ui-serif, serif;
      --font-body: var(--font-sans), ui-sans-serif, sans-serif;
      --font-mono: var(--font-code), ui-monospace, monospace;
    }

    @theme inline {
      --font-display: var(--font-heading), ui-serif, serif;
      --font-body: var(--font-sans), ui-sans-serif, sans-serif;
      --font-mono: var(--font-code), ui-monospace, monospace;
    }
    ```

    **Base styles** (after @theme):

    ```css
    /* Base element styles */
    html {
      scroll-behavior: smooth;
    }

    body {
      font-family: var(--font-body);
      font-size: var(--token-text-base);
      line-height: 1.6;
      color: var(--token-text-primary);
      background-color: var(--token-bg-primary);
      -webkit-font-smoothing: antialiased;
      -moz-osx-font-smoothing: grayscale;
    }
    ```

    IMPORTANT ordering: `:root` block MUST come BEFORE `@import "tailwindcss"` and `@theme` so that CSS custom properties are defined before Tailwind tries to resolve var() references. If Tailwind v4 requires @import first, then place the `:root` block immediately after the @import but before @theme.

    Per D-01: Dark mode is the default. No `[data-theme="light"]` overrides in this phase -- that is Phase 5 (DSGN-02/DSGN-03).

    Per D-02: Spacious whitespace -- the spacing tokens default to lg/xl for component padding. This is a value directive for component builders in later phases, not something enforced in the CSS file itself.

    Per D-03: Restrained palette -- only one accent color (oklch(0.72 0.12 178), a muted teal) plus semantic colors for status badges. No vivid rainbows.

    **Create `src/layouts/BaseLayout.astro`:**

    ```astro
    ---
    import { Font } from "astro:assets";
    import "../styles/global.css";

    interface Props {
      title?: string;
      description?: string;
    }

    const {
      title = "Jack Cutrara",
      description = "Software engineer portfolio showcasing projects and technical ability.",
    } = Astro.props;
    ---

    <html lang="en">
      <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <meta name="description" content={description} />
        <title>{title}</title>
        <Font cssVariable="--font-heading" />
        <Font cssVariable="--font-sans" />
        <Font cssVariable="--font-code" />
      </head>
      <body class="bg-bg-primary text-text-primary font-body min-h-screen antialiased">
        <slot />
      </body>
    </html>
    ```

    Key points:
    - Imports `Font` from `astro:assets` (Astro 6 Fonts API)
    - Imports `global.css` which has the design tokens + Tailwind
    - Three `<Font />` components in `<head>` -- one per font family matching the cssVariable names in astro.config.mjs
    - `<body>` uses Tailwind utility classes that resolve through the @theme bridge: `bg-bg-primary` -> `var(--token-bg-primary)` -> `oklch(0.13 0.005 260)` (near-black), `text-text-primary` -> off-white, `font-body` -> Instrument Sans
    - Props accept `title` and `description` for per-page SEO (Phase 2 will add full SEO component)
    - `min-h-screen` ensures dark background fills viewport even on short pages
    - `antialiased` for font smoothing
