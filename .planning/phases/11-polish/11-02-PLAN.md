**Step 4: WCAG AA contrast verification (D-04, QUAL-05).**

Calculate contrast ratios for every text/background combination used in the site. Use the WCAG 2.1 relative luminance formula or a CLI tool.

The combinations to check (hex values from src/styles/global.css :root):

| Text Color | Background | Usage | Hex Pair | Required Ratio | Expected |
|-----------|-----------|-------|----------|----------------|----------|
| `--ink` (#0A0A0A) | `--bg` (#FAFAF7) | Body text | #0A0A0A on #FAFAF7 | 4.5:1 (normal) | ~18.5:1 PASS |
| `--ink-muted` (#52525B) | `--bg` (#FAFAF7) | Secondary text, meta labels | #52525B on #FAFAF7 | 4.5:1 (normal) | ~6.4:1 PASS |
| `--ink-faint` (#A1A1AA) | `--bg` (#FAFAF7) | Tertiary text, captions | #A1A1AA on #FAFAF7 | 3:1 (large text) | ~3.2:1 — verify usage is large-text-only |
| `--accent` (#E63946) | `--bg` (#FAFAF7) | Links, status dot label | #E63946 on #FAFAF7 | 4.5:1 (normal) OR 3:1 (large) | ~4.0:1 — check carefully |

For the `--ink-faint` (#A1A1AA) usage: verify it is ONLY used on large text (>= 18pt or >= 14pt bold). Check the `.label-mono` class (0.75rem = 12px, but uppercase with 500 weight), `.meta-mono` class (0.8125rem = 13px), footer text. If any `--ink-faint` text is smaller than 18pt regular / 14pt bold, it may fail WCAG AA for normal text (needs 4.5:1, only has ~3.2:1). Document the finding.

For the `--accent` (#E63946) usage: if the ratio is between 3:1 and 4.5:1, it passes for large text but may fail for normal-sized links. Check where accent is used as text color at normal sizes. The accent is used for:
- Status dot label (likely large text context — OK)
- Link hover states (`.contact-link:hover`, `.footer-social-link:hover`) — these are small mono text
- Chat starter chips hover — small mono text
- `.chat-bot-message a` — inline links in chat

If accent fails 4.5:1 on normal text, document the finding and note whether it is actionable (changing the accent would break the entire design system).
