**D-05: Fix lightning-css warnings (4x "Unexpected token Delim('*')").**

The warnings are caused by Tailwind v4 auto-detecting files in `.planning/` and `design-system/` directories. These contain literal class-like strings such as `px-[var(--token-space-*)]` that lightning-css cannot parse.

In `src/styles/global.css`, add two `@source not` directives immediately AFTER the existing `@source` line at line 37. The result should be:

```css
/* Scope Tailwind class detection to source files only.
 * Tailwind v4 auto-detects template files in the repo. The .planning/ directory
 * contains v1.0 plan artifacts with literal class examples like px-[var(--token-*)]
 * that generate lightning-css parse warnings. The @source not directives exclude
 * .planning/ and design-system/ from Tailwind's detection surface. */
@source "../**/*.{astro,html,js,jsx,ts,tsx,md,mdx}";
@source not "../../.planning/**";
@source not "../../design-system/**";
```

Replace lines 32-37 with the above block (new comment + original @source + two @source not lines).
